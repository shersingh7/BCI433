1.0 概要

このガイドでは、この製品を単一ユーザー向けにローカル・ディスク・ドライブにデプロイする方法について説明します。

この製品は、他にも次のようなさまざまな方法でデプロイできます。

  • 複数のユーザーがアクセスできる共有の場所。
    IBM i Access Client Solutions: Customization and deployment made easy を参照してください。
  • ポータブル・メディア (例: CD、DVD、USB フラッシュ・ドライブ) などの選択した任意の場所、またはローカル・ディスク・ドライブ上の任意のパスに、zip アーカイブを解凍する。Documentation フォルダーにある GettingStarted 文書の 『インストール』セクションを参照してください。
  • 同じ構成を複数のユーザーに伝搬するための「構成設定の自動インポート」、および IBM i 上でコマンド行プラグインを直接使用するための「ネイティブ IBM i デプロイメント」などのオプションについては、GettingStarted 内の『デプロイメントに関するその他のオプション』を参照してください。

この製品を単一ユーザー向けにローカル・ディスク・ドライブにデプロイする場合は、本書のこれ以降のセクションを参照してください。

2.0 インストール (Windows の場合)

製品 zip アーカイブまたは製品 CD を見つけてください。

製品 zip アーカイブを使用した場合は、その zip アーカイブの内容を抽出してください。
推奨: この処置は、Windows エクスプローラで、その zip アーカイブを選択し、次いでそれを右クリックして「すべて展開(T)...」を選択することによって完了できます。

Windows エクスプローラを使用して、製品の Windows_Application フォルダーを見つけてください。

現行ユーザーに 32 ビット・バージョンをインストールする場合は、install_acs_32 をダブルクリックします。

現行ユーザーに 64 ビット・バージョンをインストールする場合は、install_acs_64 をダブルクリックします。

マルチユーザー PC 上のすべてのユーザーに 32 ビット・バージョンをインストールする場合は、install_acs_32_allusers をダブルクリックします。

マルチユーザー PC 上のすべてのユーザーに 64 ビット・バージョンをインストールする場合は、install_acs_64_allusers をダブルクリックします。

これが新規インストールの場合は、初期インストールをカスタマイズするのに役立ついくつかのプロンプトが表示されます。ご使用条件が表示された場合は、それに同意してください。

これがアップグレードの場合は、前の設定が保持されます。

インストールが終了すると、完了メッセージが表示されます。

単一ユーザーのインストールが正常に行われると、アプリケーションは次の場所にコピーされます。 

    /Users/user_id/IBM/ClientSolutions/

注: これらのスクリプトの初期バージョンは、製品ファイルを「MyDocuments」からの相対パスにコピーしていました。現行バージョンは、環境変数 %HOMEDRIVE% および %HOMEPATH% に基づくパスに製品ファイルをコピーします。これらのスクリプトを使用して製品を更新する場合、更新は最初に検出された場所に対して行われます。

マルチユーザーのインストールが正常に行われると、アプリケーションは次の場所にコピーされます。 

    /Users/Public/IBM/ClientSolutions/

2.1 製品の開始 (Windows の場合)

デスクトップにある「Access Client Solutions」アイコンをダブルクリックしてください。

3.0 インストール (Mac の場合)

Finder を使用して製品 zip アーカイブまたは製品 CD を見つけてください。

製品 zip アーカイブを使用した場合は、その zip アーカイブの内容を抽出してください。
推奨: たった今製品をダウンロードしたばかりならば、「ダウンロード」の中を見てください。zip アーカイブの内容が抽出されていない場合は、zip アーカイブをダブルクリックすれば、その内容を抽出することができます。

Mac_Application フォルダーを見つけてください。 install_acs をダブルクリックします。

これが新規インストールの場合は、初期インストールをカスタマイズするのに役立ついくつかのプロンプトが表示されます。ご使用条件が表示された場合は、それに同意してください。

これがアップグレードの場合は、前の設定が保持されます。

インストールが終了すると、完了メッセージが表示されます。

インストールが正常に行われると、アプリケーションは Applications フォルダーにコピーされます。

Finder からインストールを開始した場合、インストールを実行するために端末ウィンドウが開いています。インストールが完了したら、端末ウィンドウを閉じることができます。

技術情報:
セキュリティー設定、および使用している OS X のバージョンによっては、install_acs の実行が許可されていないことを示すポップアップ・ウィンドウが表示されることがあります。 その場合は、次のいずれかの方法によって、install_acs の実行を許可する必要があります。

  • install_acs を「開く」オプションが提示された場合は、それを選択します。
  • install_acs を右クリックし、「開く」を選択します。
    ポップアップ・ウィンドウが提示されたら、「開く」を選択してください。
  • 上記のオプションがどちらも機能しない場合は、以下の選択を行います。
    システム環境設定...
    セキュリティとプライバシー
    「IBM i Access Client Solutions」を開くためのオプションがある場合は、それを選択します。オプションがない場合は、「一般」タブの設定を調整して、他のアプリケーションを実行できるようにする必要があります。「IBM i Access Client Solutions」が初めて開始されたときは、開始後にセキュリティー設定を元に戻すことができます。

3.1 製品の開始 (Mac の場合)

Finder を使用して Applications に移動してください。

「IBM i Access Client Solutions」を見つけてください。見つかったら、それをダブルクリックして製品を開始してください。

技術情報:
セキュリティー設定、および使用している OS X のバージョンによっては、アプリケーションの実行が許可されていないことを示すポップアップ・ウィンドウが表示されることがあります。 その場合は、次のいずれかの方法によって、アプリケーションの実行を許可する必要があります。

  • アプリケーションを「開く」オプションが提示された場合は、それを選択します。
  • 「IBM i Access Client Solutions」アイコンを右クリックし、「開く」を選択します。
    ポップアップ・ウィンドウが提示されたら、「開く」を選択してください。
  • 上記のオプションがどちらも機能しない場合は、以下の選択を行います。
    システム環境設定...
    セキュリティとプライバシー
    「IBM i Access Client Solutions」を開くためのオプションがある場合は、それを選択します。オプションがない場合は、「一般」タブの設定を調整して、他のアプリケーションを実行できるようにする必要があります。「IBM i Access Client Solutions」が初めて開始されたときは、開始後にセキュリティー設定を元に戻すことができます。
技術情報:
  • 「Java 6 SE ランタイム」のインストールを指示するプロンプトが出されても、Java 6 をインストールする必要はありません。 Java 8 をインストールして、製品の開始を再試行してください。
  • Java を Mac にインストールする場合は、JRE でなく JDK を選択してください。Mac 用の JRE では、Java がブラウザー用にのみインストールされます。 他のアプリケーション用には Java がインストールされません。JDK をインストールすれば、IBM i Access Client Solutions など、他のアプリケーションで Java を使用できるようになります。

4.0 インストール (Linux の場合)

製品 zip アーカイブまたは製品 CD を見つけてください。

製品 zip アーカイブを使用した場合は、その zip アーカイブの内容を抽出してください。
推奨: 端末ウィンドウから unzip コマンドを使用してください。

Linux_Application フォルダーを見つけてください。

インストール・スクリプトを使用するには root 権限が必要なので、sudo を使用して端末ウィンドウからインストール・スクリプトを実行してください。 例:

32 ビット・アーキテクチャーの場合は、次のコマンドを使用します。

    sudo ./install_acs_32
64 ビット・アーキテクチャーの場合は、次のコマンドを使用します。
    sudo ./install_acs_64

これが新規インストールの場合は、初期インストールをカスタマイズするのに役立ついくつかのプロンプトが表示されます。ご使用条件が表示された場合は、それに同意してください。

これがアップグレードの場合は、前の設定が保持されます。

インストールが終了すると、完了メッセージが表示されます。

インストールが正常に行われると、アプリケーションは次の場所にコピーされます。

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 製品の開始 (Linux の場合)

アプリケーション・フォルダー内で「IBM i Access Client Solutions」を検索するか、Super キーを使用してから「IBM i Access Client Solutions」と入力し、検索します。

5.0 アップデートのインストール

新しいバージョンが使用可能になったら、この文書に示されているインストール手順に従ってアップデートを適用できます。 既存のファイルの上書きを求めるプロンプトが出されたら、「すべてはい」で答えてください。

Windows、Mac、または Linux の install_acs スクリプトを使用した場合は、AcsConfig.properties ファイルは更新されません。これは、このファイルに対して行ったカスタム構成変更が失われないようにするためです。

6.0 詳細情報

製品のカスタマイズに関する追加情報が必要な場合や、製品のインストール中または開始中に問題に遭遇した場合は、Documentation フォルダーにある GettingStarted 文書を参照してください。

7.0 アンインストール

インストール・スクリプトによって上記の場所にコピーされたアプリケーション・ファイルを削除するには、インストール・スクリプトと同じフォルダー内にある、以下のプラットフォーム固有のアンインストール・スクリプトを使用します。 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
