1.0 Introduction

Ce guide explique comment déployer ce produit sur une unité de disque locale pour un seul utilisateur.

Il existe un certain nombre d'autres manières de déployer ce produit, notamment :

  • En utilisant une unité réseau partagée accessible par plusieurs utilisateurs.
    Voir IBM i Access Client Solutions: Customization and deployment made easy
  • En décompressant l'archive ZIP n'importe où sur le support amovible de votre choix (par exemple, CD-ROM, DVD-ROM, clé USB) ou dans n'importe quel chemin de l'unité de disque locale. Voir la section Installation du document GettingStarted situé dans le dossier Documentation.
  • Voir la section Autres options de déploiement dans le document Guide de démarrage rapide pour en savoir plus sur les options telles que Importation automatique de paramètres de configuration permettant de propager la même configuration à plusieurs utilisateurs et l'option Déploiements natifs sur l'IBM i permettant d'utiliser les modules d'extension de ligne de commande directement sur l'IBM i.

Pour déployer ce produit sur l'unité de disque locale pour un seul utilisateur, voir les autres sections de ce document.

2.0 Installation pour Windows

Recherchez l'archive zip ou le CD du produit.

Si vous utilisez l'archive zip du produit, il faut extraire son contenu.
Suggestion : dans l'explorateur Windows, sélectionnez l'archive zip, puis cliquez avec le bouton droit de la souris et choisissez "Extraire tout...".

Dans l'explorateur Windows, localisez le dossier Windows_Application du produit.

Pour installer la version 32 bits pour l'utilisateur en cours, cliquez deux fois sur install_acs_32.

Pour installer la version 64 bit pour l'utilisateur en cours, cliquez deux fois sur install_acs_64.

Pour installer la version 32 bits pour tous les utilisateurs d'un ordinateur multiutilisateur, cliquez deux fois sur install_acs_32_allusers.

Pour installer la version 64 bits pour tous les utilisateurs d'un ordinateur multiutilisateur, cliquez deux fois sur install_acs_64_allusers.

S'il s'agit d'une nouvelle installation, plusieurs invites s'afficheront pour vous aider à personnaliser l'installation initiale. Si un contrat de licence s'affiche, acceptez-le.

S'il s'agit d'une mise à niveau, les paramètres précédents seront conservés.

A la fin de l'installation, un message d'achèvement s'affiche.

Lorsqu'une installation monoutilisateur aboutit, l'application est copiée dans : 

    /Users/user_id/IBM/ClientSolutions/

Remarque : La version initiale de ces scripts copiait les fichiers de produit dans un chemin relatif à MyDocuments. La version en cours copie les fichiers de produit dans un chemin basé sur les variables d'environnement %HOMEDRIVE% et %HOMEPATH%. Lorsque ces scripts sont utilisés pour mettre à jour le produit, des mise à jour sont effectuées sur le premier emplacement trouvé.

Lorsqu'une installation multiutilisateur aboutit, l'application est copiée dans : 

    /Users/Public/IBM/ClientSolutions/

2.1 Démarrage du produit sous Windows

Double-cliquez sur l'icône "Access Client Solutions" sur le bureau.

3.0 Installation pour Mac

Recherchez l'archive zip ou le CD du produit à l'aide de Finder.

Si vous utilisez l'archive zip du produit, il faut extraire son contenu.
Suggestion : si vous venez de télécharger le produit, recherchez l'archive dans le dossier Téléchargements. Si son contenu n'a pas été extrait, faites-le en double-cliquant dessus.

Recherchez le dossier Mac_Application. Cliquez deux fois sur install_acs

S'il s'agit d'une nouvelle installation, plusieurs invites s'afficheront pour vous aider à personnaliser l'installation initiale. Si un contrat de licence s'affiche, acceptez-le.

S'il s'agit d'une mise à niveau, les paramètres précédents seront conservés.

A la fin de l'installation, un message d'achèvement s'affiche.

Lorsqu'une installation aboutit, l'application est copiée dans le dossier Applications.

Si vous avez démarré l'installation à partir de Finder, une fenêtre de terminal s'est ouverte pour vous permettre d'effectuer l'installation. Une fois l'installation terminée, vous pouvez fermer la fenêtre de terminal.

Remarque technique :
Selon vos paramètres de sécurité et la version d'OS X que vous utilisez, il se peut qu'une fenêtre en incrustation indiquant que install_acs n'est pas autorisé à s'exécuter s'affiche. Dans ce cas, vous devez autoriser install_acs à s'exécuter en procédant de l'une des façons suivantes :

  • Si une option "Ouvrir" vous est proposée, sélectionnez cette option.
  • Cliquez avec le bouton droit de la souris sur install_acs et sélectionnez "Ouvrir".
    Lorsqu'une fenêtre en incrustation s'affiche, sélectionnez "Ouvrir".
  • Si aucune des options ci-dessus ne fonctionne, accédez à :
    Préférences du système...
    Sécurité et confidentialité
    S'il existe une option permettant d'ouvrir "IBM i Access Client Solutions", sélectionnez-la. Sinon, vous devrez ajuster les paramètres dans l'onglet Général afin d'autoriser les autres applications à s'exécuter. Une fois qu'"IBM i Access Client Solutions" a démarré une première fois, vous pouvez revenir à vos paramètres de sécurité d'origine.

3.1 Démarrage du produit sur Mac

Accédez à Applications à l'aide de Finder.

Recherchez "IBM i Access Client Solutions" et cliquez deux fois dessus pour démarrer le produit.

Remarque technique :
Selon vos paramètres de sécurité et la version d'OS X que vous utilisez, il se peut qu'une fenêtre en incrustation indiquant que l'application n'est pas autorisée à s'exécuter s'affiche. Dans ce cas, vous devez autoriser l'application à s'exécuter en procédant de l'une des façons suivantes :

  • Si une option "Ouvrir" vous est proposée, sélectionnez cette option.
  • Cliquez avec le bouton droit de la souris sur l'icône "IBM i Access Client Solutions" et sélectionnez "Ouvrir".
    Lorsqu'une fenêtre en incrustation s'affiche, sélectionnez "Ouvrir".
  • Si aucune des options ci-dessus ne fonctionne, accédez à :
    Préférences du système...
    Sécurité et confidentialité
    S'il existe une option permettant d'ouvrir "IBM i Access Client Solutions", sélectionnez-la. Sinon, vous devrez ajuster les paramètres dans l'onglet Général afin d'autoriser les autres applications à s'exécuter. Une fois qu'"IBM i Access Client Solutions" a démarré une première fois, vous pouvez revenir à vos paramètres de sécurité d'origine.
Remarque technique :
Si vous êtes invité à installer "Java 6 SE runtime", vous n'avez pas besoin d'installer Java 6. Installez Java 8, puis réessayez de démarrer le produit.

4.0 Installation pour Linux

Recherchez l'archive zip ou le CD du produit.

Si vous utilisez l'archive zip du produit, il faut extraire son contenu.
Suggestion : Depuis une fenêtre de terminal, utilisez la commande unzip.

Recherchez le dossier Linux_Application.

Etant donné que vous devrez disposer des droits d'accès root pour utiliser le script d'installation, utilisez sudo pour exécuter le script d'installation à partir d'une fenêtre de terminal. Par exemple :

Si vous êtes sur une architecture 32 bits, utilisez la commande suivante :

    sudo ./install_acs_32
Si vous êtes sur une architecture 64 bits, utilisez la commande suivante :
    sudo ./install_acs_64

S'il s'agit d'une nouvelle installation, plusieurs invites s'afficheront pour vous aider à personnaliser l'installation initiale. Si un contrat de licence s'affiche, acceptez-le.

S'il s'agit d'une mise à niveau, les paramètres précédents seront conservés.

A la fin de l'installation, un message d'achèvement s'affiche.

Lorsqu'une installation aboutit, l'application est copiée dans :

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Démarrage du produit sous Linux

Recherchez "IBM i Access Client Solutions" dans le dossier des applications ou utilisez la touche Super/Windows, puis tapez "IBM i Access Client Solutions".

5.0 Installations de mises à jour

Lorsqu'une nouvelle version est disponible, vous pouvez suivre la procédure d'installation des mises à jour décrite dans ce document. Si vous êtes invité à écraser les fichiers existants, répondez "Oui à tout".

Si vous utilisez les scripts install_acs pour Windows, Mac ou Linux, le fichier AcsConfig.properties n'est pas mis à jour. Par conséquent, toutes les modifications de configuration personnalisées que vous avez apportées à ce fichier ne sont pas perdues.

6.0 Informations complémentaires

Pour plus d'informations sur la personnalisation du produit ou si vous êtes confronté à des problèmes lors du démarrage du produit, reportez-vous au document GettingStarted (Mise en route) situé dans le répertoire Documentation.

7.0 Désinstallation

Pour supprimer les fichiers d'application qui ont été copiés aux emplacements ci-dessus par le script d'installation, utilisez le script de désinstallation spécifique à la plateforme, qui se trouve dans le même dossier que le script d'installation : 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
