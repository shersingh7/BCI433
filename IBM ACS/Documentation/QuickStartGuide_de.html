1.0 Einführung

Dieser Schnelleinstieg enthält Anweisungen zur Implementierung dieses Produkts auf einer lokale Festplatte für einen einzelnen Benutzer.

Es gibt eine Vielzahl von anderen Möglichkeiten, wie dieses Produkt bereitgestellt werden kann. Dazu gehören:

  • Sie verwenden eine gemeinsam genutzte Position, auf die mehrere Benutzer zugreifen können.
    Weitere Informationen finden Sie im Artikel IBM i Access Client Solutions: Customization and deployment made easy.
  • Sie entpacken das ZIP-Archiv an einer beliebigen Position Ihrer Wahl, die auch ein tragbares Medium (z. B. CD, DVD oder USB-Flashlaufwerk) oder ein beliebiger Pfad im lokalen Festplattenlaufwerk sein kann. Weitere Informationen enthält der Abschnitt Installation des Dokuments Erste Schritte, das im Dokumentationsordner verfügbar ist.
  • Im Abschnitt Andere Implementierungsoptionen des Dokuments Erste Schritte finden Sie beispielsweise unter Konfigurationseinstellungen automatisch importieren Informationen dazu, wie Sie dieselbe Konfiguration an mehrere Benutzer weitergeben können. Unter Native IBM i-Implementierungen ist dort beschrieben, wie Sie die Befehlszeilen-Plug-ins direkt auf dem IBM i-System verwenden.

Die Implementierung dieses Produkts auf einer lokale Festplatte für einen einzelnen Benutzer ist in den übrigen Abschnitten des vorliegenden Dokuments beschrieben.

2.0 Installation für Windows

Suchen Sie das ZIP-Archiv oder die Produkt-CD.

Wenn Sie das ZIP-Archiv des Produkts verwenden, extrahieren Sie jetzt dessen Inhalt.
Vorschlag: Dazu können Sie folgendermaßen vorgehen: Wählen Sie das ZIP-Archiv mit Windows Explorer aus, klicken Sie mit der rechten Maustaste darauf, und wählen Sie dann "Alle extrahieren..." aus.

Suchen Sie mit Windows Explorer den Windows_Application-Ordner des Produkts aus.

Zur Installation der 32-Bit-Version für den aktuellen Benutzer doppelklicken Sie auf "install_acs_32".

Zur Installation der 32-Bit-Version für den aktuellen Benutzer doppelklicken Sie auf "install_acs_64".

Zur Installation der 32-Bit-Version für alle Benutzer auf einem PC für mehrere Benutzer doppelklicken Sie auf "install_acs_32_allusers".

Zur Installation der 64-Bit-Version für alle Benutzer auf einem PC für mehrere Benutzer doppelklicken Sie auf "install_acs_64_allusers".

Bei einer Neuinstallation werden mehrere Eingabeaufforderungen angezeigt, in denen Sie die Erstinstallation anpassen können. Falls eine Lizenzvereinbarung angezeigt wird, akzeptieren Sie diese.

Bei einem Upgrade bleiben die bisherigen Einstellungen erhalten.

Nach Abschluss der Installation wird eine Beendigungsnachricht angezeigt.

Bei einer erfolgreichen Installation für einen Einzelbenutzer wird die Anwendung an die folgende Position kopiert: 

    /Users/benutzer-id/IBM/ClientSolutions/

Anmerkung: Bei der ersten Version dieser Scripts wurden die Produktdateien in einen Pfad in Bezug zu MyDocuments kopiert. Bei der aktuellen Version werden die Produktdateien auf der Basis der Umgebungsvariablen %HOMEDRIVE% und %HOMEPATH% in einen Pfad kopiert. Werden diese Scripts zur Aktualisierung des Produkts verwendet, finden Aktualisierungen an der zuerst gefundenen Position statt.

Bei einer erfolgreichen Installation für mehrere Benutzer wird die Anwendung an die folgende Position kopiert: 

    /Users/Public/IBM/ClientSolutions/

2.1 Produkt unter Windows starten

Doppelklicken Sie auf das Symbol "Access Client Solutions" auf dem Desktop.

3.0 Installation für Mac

Suchen Sie das ZIP-Archiv oder die CD des Produkts mit dem Finder.

Wenn Sie das ZIP-Archiv des Produkts verwenden, extrahieren Sie jetzt dessen Inhalt.
Vorschlag: Wenn Sie das Produkt gerade heruntergeladen haben, suchen Sie unter "Downloads". Sofern der Inhalt des ZIP-Archivs noch nicht extrahiert wurde, können sie ihn jetzt durch Doppelklicken auf das ZIP-Archiv extrahieren.

Suchen Sie den Ordner "Mac_Application". Doppelklicken Sie auf "install_acs".

Bei einer Neuinstallation werden mehrere Eingabeaufforderungen angezeigt, in denen Sie die Erstinstallation anpassen können. Falls eine Lizenzvereinbarung angezeigt wird, akzeptieren Sie diese.

Bei einem Upgrade bleiben die bisherigen Einstellungen erhalten.

Nach Abschluss der Installation wird eine Beendigungsnachricht angezeigt.

Bei einer erfolgreichen Installation wird die Anwendung in den Ordner "Applications" kopiert.

Falls Sie die Installation aus der Funktion "Finder" heraus gestartet haben, wurde zur Durchführung der Installation ein Terminalfenster geöffnet. Nach Abschluss der Installation können Sie das Terminalfenster schließen.

Technischer Hinweis:
Abhängig von Ihren Sicherheitseinstellungen und der verwendeten Version des Betriebssystems X erscheint möglicherweise ein Popup-Fenster mit dem Hinweis, dass keine Berechtigung für die Ausführung von "install_acs" vorhanden ist. Ist dies der Fall, müssen Sie eine Berechtigung für die Ausführung von "install_acs" erteilen. Verwenden Sie dazu eine der folgenden Methoden:

  • Wenn Ihnen eine Option zum Öffnen von "install_acs" angeboten wird, wählen Sie sie aus.
  • Klicken Sie mit der rechten Maustaste auf "install_acs" und wählen Sie "Open" (Öffnen) aus.
    Wenn ein Popup-Fenster erscheint, wählen Sie "Open" aus.
  • Wenn keine der beiden oben genannten Optionen funktioniert, rufen Sie Folgendes auf:
    System Preferences...
    Security & Privacy
    Wenn eine Option zum Öffnen von IBM i Access Client Solutions verfügbar ist, wählen Sie diese Option aus. Andernfalls müssen Sie die Einstellungen auf der Registerkarte "General" (Allgemein) anpassen, damit die Ausführung anderer Anwendungen zulässig ist. Nachdem "IBM i Access Client Solutions" zum ersten Mal gestartet wurde, können Sie Ihre Sicherheitseinstellungen zurückändern.

3.1 Produkt auf Mac starten

Rufen Sie mit dem Finder "Applications" auf.

Suchen Sie "IBM i Access Client Solutions" und doppelklicken Sie darauf, um das Produkt zu starten.

Technischer Hinweis:
Abhängig von Ihren Sicherheitseinstellungen und der verwendeten Version des Betriebssystems X erscheint möglicherweise ein Popup-Fenster mit dem Hinweis, dass keine Berechtigung für die Ausführung der Anwendung vorhanden ist. Ist dies der Fall, müssen Sie eine Berechtigung für die Ausführung der Anwendung erteilen. Verwenden Sie dazu eine der folgenden Methoden:

  • Wenn Ihnen eine Option zum Öffnen der Anwendung angeboten wird, wählen Sie sie aus.
  • Klicken Sie mit der rechten Maustaste auf das Symbol "IBM i Access Client Solutions" und wählen Sie "Open" aus.
    Wenn ein Popup-Fenster erscheint, wählen Sie "Open" aus.
  • Wenn keine der beiden oben genannten Optionen funktioniert, rufen Sie Folgendes auf:
    System Preferences...
    Security & Privacy
    Wenn eine Option zum Öffnen von IBM i Access Client Solutions verfügbar ist, wählen Sie diese Option aus. Andernfalls müssen Sie die Einstellungen auf der Registerkarte "General" (Allgemein) anpassen, damit die Ausführung anderer Anwendungen zulässig ist. Nachdem "IBM i Access Client Solutions" zum ersten Mal gestartet wurde, können Sie Ihre Sicherheitseinstellungen zurückändern.
Technischer Hinweis:
Wenn Sie zur Installation von "Java 6 SE runtime" aufgefordert werden, müssen Sie Java 6 nicht installieren. Installieren Sie Java 8 und versuchen Sie dann, das Produkt erneut zu starten.

4.0 Installation für Linux

Suchen Sie das ZIP-Archiv oder die Produkt-CD.

Wenn Sie das ZIP-Archiv des Produkts verwenden, extrahieren Sie jetzt dessen Inhalt.
Vorschlag: Verwenden Sie den unzip-Befehl im Terminalfenster.

Suchen Sie den Ordner "Linux_Application".

Da Sie für das Installationsscript Rootberechtigung benötigen, müssen Sie sudo verwenden, um das Installationsscript in einem Terminalfenster auszuführen. Beispiel:

Verwenden Sie bei einer 32-Bit-Architektur den folgenden Befehl: 

    sudo ./install_acs_32
Verwenden Sie bei einer 64-Bit-Architektur den folgenden Befehl: 
    sudo ./install_acs_64

Bei einer Neuinstallation werden mehrere Eingabeaufforderungen angezeigt, in denen Sie die Erstinstallation anpassen können. Falls eine Lizenzvereinbarung angezeigt wird, akzeptieren Sie diese.

Bei einem Upgrade bleiben die bisherigen Einstellungen erhalten.

Nach Abschluss der Installation wird eine Beendigungsnachricht angezeigt.

Bei einer erfolgreichen Installation wird die Anwendung an die folgende Position kopiert: 

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Produkt unter Linux starten

Sie können nach "IBM i Access Client Solutions" im Anwendungsordner suchen oder indem Sie die Super-Taste verwenden und dann "IBM i Access Client Solutions" eingeben.

5.0 Installation von Updates

Wenn eine neue Version verfügbar ist, können Sie die Aktualisierungen anhand der Installationsanweisungen in diesem Dokument ausführen. Die Aufforderung, vorhandene Dateien zu überschreiben, beantworten Sie mit "Yes to all".

Wenn sie die Scripts "install_acs" für Windows, Mac oder Linux verwenden, wird die Datei AcsConfig.properties nicht aktualisiert, dadurch gehen keine der von Ihnen vorgenommenen Änderungen der angepassten Konfiguration verloren.

6.0 Weitere Informationen

Weitere Informationen zur Anpassung des Produkts oder Hinweise, wenn Probleme bei der Installation oder dem Starten des Produkts auftreten, finden Sie im Dokument Erste Schritte, das sich im Verzeichnis der Produktdokumentation befindet.

7.0 Deinstallation

Um die Anwendungsdateien, die von dem Installationsscript an die oben genannten Positionen kopiert wurden, zu entfernen, verwenden Sie das plattformspezifische Deinstallationsscript, das sich in demselben Ordner wie das Installationsscript befindet. 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
