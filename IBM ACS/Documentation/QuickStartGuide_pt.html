1.0 Introdução

Este manual fornece instruções sobre como implementar este produto numa unidade de disco local para um único utilizador.

Existe uma variedade de outras formas nas quais também é possível implementar este produto, incluindo:

  • Uma localização partilhada que pode ser acedida por múltiplos utilizadores.
    Consulte IBM i Access Client Solutions: Personalização e implementação facilitadas
  • Descompacte o arquivo Zip para a localização que pretender, incluindo suportes portáteis (ex, CD, DVD e unidades USB) ou para qualquer caminho na unidade de disco local. Consulte a secção Instalação do documento Introdução localizado na pasta Documentação.
  • Consulte Outras Opções de Implementação na Introdução para obter opções sobre como Importar automaticamente definições de configuração, para passar a mesma configuração para múltiplos utilizadores, e Implementações Nativas do IBM i para utilizar suplementos directos de linha de comandos no IBM i.

Para implementar este produto numa unidade de disco local para um único utilizador, consulte as restantes secções deste documento.

2.0 Instalação para Windows

Localize o arquivo ZIP ou o CD do produto.

Se está a utilizar o arquivo zip do produto, extraia o conteúdo do arquivo zip.
Sugestão: Pode fazê-lo através do Windows Explorer, seleccione o ficheiro e, em seguida, faça clique com o botão direito sobre o ficheiro e por fim seleccione "Extrair todos...".

Com o Windows Explorer, localize a pasta Windows_Application do produto.

Para instalar a versão de 32 bit para o utilizador actual, faça duplo clique sobre install_acs_32.

Para instalar a versão de 64 bit para o utilizador actual, faça duplo clique sobre install_acs_64.

Para instalar a versão de 32 bit para todos os utilizadores num PC de múltiplos utilizadores, faça duplo clique em install_acs_32_allusers.

Para instalar a versão de 64 bit para todos os utilizadores num PC de múltiplos utilizadores, faça duplo clique em install_acs_64_allusers.

Se esta é uma nova versão, existirão vários pedidos que o ajudarão a personalizar a instalação inicial. Se aparecer um acordo de licença, aceite-o.

Se esta for uma actualização, as definições anteriores têm de ser preservadas.

Quando a instalação estiver concluída, é apresentada uma mensagem de conclusão.

Uma instalação de utilizador único bem sucedida irá copiar a aplicação para: 

    /Users/user_id/IBM/ClientSolutions/

Nota: A versão inicial destes scripts copiaram os ficheiros do produto para um caminho relativo a Os meus documentos (MyDocuments). A versão actual copia os ficheiros do produto para um caminho baseado nas variáveis de ambiente %HOMEDRIVE% e %HOMEPATH%. Quando utiliza estes scripts para actualizar o produto, as actualizações ocorrerão na primeira localização encontrada.

Uma instalação de múltiplos utilizadores bem sucedida irá copiar a aplicação para: 

    /Users/Public/IBM/ClientSolutions/

2.1 Iniciar o produto no Windows

Faça duplo clique no ícone de "Access Client Solutions" no ambiente de trabalho.

3.0 Instalação para Mac

Localize o arquivo ZIP ou o CD do produto utilizando o Localizador.

Se está a utilizar o arquivo zip do produto, extraia o conteúdo do arquivo zip.
Sugestão: Se tiver acabado de descarregar o produto, procure em Transferências. Se o conteúdo do arquivo ZIP não tiver sido extraído, então extraia o conteúdo do arquivo ZIP, para isso faça duplo clique sobre o arquivo.

Localize a pasta Mac_Application. Faça duplo clique em install_acs

Se esta é uma nova versão, existirão vários pedidos que o ajudarão a personalizar a instalação inicial. Se aparecer um acordo de licença, aceite-o.

Se esta for uma actualização, as definições anteriores têm de ser preservadas.

Quando a instalação estiver concluída, é apresentada uma mensagem de conclusão.

Uma instalação bem sucedida irá copiar a aplicação para a pasta Aplicações.

Se iniciou a instalação a partir do Localizador, foi aberta uma janela de terminal para efectuar a instalação. Quando a instalação estiver concluída, poderá fechar a janela do terminal.

Nota Técnica:
Dependendo das definições de segurança e a versão do OS X que está a utilizar, poderá surgir uma janela emergente que indica que install_acs não tem autorização para executar. Caso isso aconteça, será necessário autorizar install_acs a executar através da utilização de um dos seguintes métodos:

  • Se for apresentada uma opção para "Abrir" install_acs, seleccione-a.
  • Faça clique com o botão direito do rato em install_acs e seleccione "Abrir".
    Quando surgir uma janela emergente, seleccione "Abrir".
  • Se nenhuma das opções acima funcionar, aceda a:
    Preferências do Sistema...
    Segurança & Privacidade
    Se existir uma opção para Abrir o "IBM i Access Client Solutions", seleccione-a. Caso contrário, será necessário ajustar as definições no separador Geral para permitir que outras aplicações sejam executadas. Após "IBM i Access Client Solutions" iniciar pela primeira vez, poderá voltar a alterar as definições de segurança.

3.1 Iniciar o produto em Mac

Utilize o Localizador para aceder a Aplicações.

Localize "IBM i Access Client Solutions" e faça duplo clique para iniciar o produto.

Nota Técnica:
Dependendo das definições de segurança e a versão do OS X que está a utilizar, poderá surgir uma janela emergente que indica que a aplicação não tem autorização para executar. Caso isso aconteça, será necessário autorizar a aplicação a executar através da utilização de um dos seguintes métodos:

  • Se for apresentada uma opção para "Abrir" a aplicação, seleccione-a.
  • Faça clique com o botão direito do rato no ícone "IBM i Access Client Solutions" e seleccione "Abrir".
    Quando surgir uma janela emergente, seleccione "Abrir".
  • Se nenhuma das opções acima funcionar, aceda a:
    Preferências do Sistema...
    Segurança & Privacidade
    Se existir uma opção para Abrir o "IBM i Access Client Solutions", seleccione-a. Caso contrário, será necessário ajustar as definições no separador Geral para permitir que outras aplicações sejam executadas. Após "IBM i Access Client Solutions" iniciar pela primeira vez, poderá voltar a alterar as definições de segurança.
Nota Técnica:
Caso seja pedido para instalar "Java 6 SE runtime", não será necessário instalar o Java 6. Instale o Java 8 e, em seguida, tente iniciar o produto novamente.

4.0 Instalação para Linux

Localize o arquivo ZIP ou o CD do produto.

Se está a utilizar o arquivo zip do produto, extraia o conteúdo do arquivo zip.
Sugestão: A partir de uma janela do terminal, utilize o comando unzip.

Localize a pasta Linux_Application.

Uma vez que será necessário ter autoridade raiz para utilizar o script de instalação, utilize sudo para executar o script de instalação a partir de uma janela do terminal. Por exemplo:

Se estiver numa arquitectura de 32 bits, utilize o seguinte comando: 

    sudo ./install_acs_32
Se estiver numa arquitectura de 64 bits, utilize o seguinte comando: 
    sudo ./install_acs_64

Se esta é uma nova versão, existirão vários pedidos que o ajudarão a personalizar a instalação inicial. Se aparecer um acordo de licença, aceite-o.

Se esta for uma actualização, as definições anteriores têm de ser preservadas.

Quando a instalação estiver concluída, é apresentada uma mensagem de conclusão.

Uma instalação bem sucedida irá copiar a aplicação para: 

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Iniciar o Produto em Linux

Procure por "IBM i Access Client Solutions" na pasta das aplicações ou através da utilização da Super key e, em seguida, escreva "IBM i Access Client Solutions".

5.0 Instalar Actualizações

Quando é disponibilizada uma nova versão, é possível seguir as instruções de instalação neste documento para aplicar as actualizações. Se for pedido para sobrepor ficheiros existentes, responda com "Sim para todos".

Se estiver a utilizar os scripts install_acs para Windows, Mac ou Linux, o ficheiro AcsConfig.properties não será actualizado e, como tal, quaisquer alterações de configuração personalizadas que tiver efectuado ao mesmo não serão perdidas.

6.0 Mais Informações

Para obter informações adicionais sobre a personalização do produto ou caso detecte problemas durante a instalação ou início do produto, consulte o documento Iniciação (GettingStarted) localizado na pasta Documentação.

7.0 Desinstalar

Para remover os ficheiros de aplicação que foram copiados para as localizações acima pelo script de instalação, utilize o script de desinstalação específico da plataforma localizado na mesma pasta que o script de instalação: 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
