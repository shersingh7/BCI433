1.0 소개

IBM i Access Client Solutions는 IBM i Access 제품군 중 최신 제품입니다. 본 제품에는 Linux, Mac 및 Windows를 포함해, Java를 지원하는 대부분의 오퍼레이팅 시스템에서 실행되는 Java 기반 플랫폼 독립 인터페이스가 제공됩니다. IBM i Access Client Solutions는 단순화된 하나의 위치에서 IBM i를 관리하기 위해 일반적으로 사용되는 태스크를 통합합니다.

IBM i Access Client Solutions는 다른 IBM i Access 제품군과 동일한 IBM i 호스트 서버를 사용하고, 동일한 IBM i Access 제품군 라이센스(XW1)가 있어야 5250 에뮬레이션 및 데이터 전송 피처를 사용할 수 있습니다.

Windows 및 Linux용 클라이언트 애플리케이션을 사용 및 개발하기 위한 미들웨어가 포함된 두 개의 선택적 패키지도 사용할 수 있습니다.
    IBM i Access Client Solutions - Windows Application Package
    IBM i Access Client Solutions - Linux Application Package

현재 자격이 있는 IBM i 7.1 이상을 사용 중인 고객은 다음 두 가지 방법 중 하나를 사용하여 IBM i Access Client Solutions을 확보할 수 있습니다.

  • 기본 IBM i Access Client Solutions 패키지를 다운로드합니다.
  • 5761-SS1 또는 5770-SS1의 ESS(Entitled Software Support) 웹 사이트에서 기본 패키지와 선택사항인 Windows 및 Linux 애플리케이션 패키지를 다운로드합니다.

선택사항인 Windows 및 Linux 애플리케이션 패키지는 ESS(Entitled Software Support) 웹 사이트를 통해서만 확보할 수 있습니다.

고객은 5761-SS1 또는 5770-SS1 화면정리 피처 6288을 주문하여 매체를 확보할 수 있습니다. 실제 매체에는 CD를 통해 제품을 바로 실행할 수 있도록 하는 제품의 실행 가능 버전이 포함되어 있습니다. 또한 실제 매체에는 복사하여 원하는 위치에 추출할 수 있는 제품의 zip 아카이브도 포함되어 있습니다. IBM i Access Client Solutions의 실제 매체에는 선택적인 Windows 및 Linux Application Package가 포함되어 있지 않습니다.

추가 정보는 다음 웹 사이트를 방문하여 확인하십시오.
    http://www.ibm.com/systems/power/software/i/access/solutions.html

IBM i Access 제품군에 대한 최신 정보는 다음 웹 사이트를 방문하십시오.
    http://www.ibm.com/systems/i/software/access/caann.html

2.0 피처

IBM i Access Client Solutions 피처에는 다음이 포함됩니다.

  • IBM Rational Host-on-Demand를 기반으로 완전한 피처를 갖춘 5250 표시장치 에뮬레이터. IBM i Access for Windows를 사용하는 경우 익숙한 5250 표시장치 피처 모두와 함께 일부 피처가 추가됩니다.
    • 다중 세션이 단일 보기 영역을 공유할 수 있는 탭 지원
    • 페이지 사이에서 즉각적으로 화명이동할 수 있는 마우스휠 지원
    • 워크스테이션을 다시 부팅하지 않고 언어 전환 가능
    • 서로 다른 에뮬레이터 세션에서 별도의 언어를 사용할 수 있는 호스트 코드 페이지가 다른 동시 표시 화면 세션
  • 프린터 에뮬레이션
  • IBM 퍼스널 통신 세션 관리자 다음에 모델링되어 5250 에뮬레이터 세션 모두를 관리하는 데 사용할 수 있는 5250 세션 관리자
  • IBM i 데이터베이스와 워크스테이션의 여러 파일 유형(예: OpenDocument 스프레드시트(*.ods), Excel Workbook(*.xlsx) 및 기타 파일 형식) 사이에서 데이터를 전송하는 기능을 제공하는 데이터 전송
  • 프린터 출력은 IBM i 출력 큐의 파일을 볼 수 있는 인터페이스를 제공하고 이러한 파일을 클라이언트 시스템에 다운로드할 수 있는 기능도 제공합니다.
  • IBM i 조작 패널에 대한 그래픽 인터페이스가 있는 가상 제어판
  • LAN, HMC 및 FSM 콘솔의 5250 에뮬레이션
  • ASMI, IVM 및 HMC를 포함한 하드웨어 관리 인터페이스 구성의 통합
  • 디폴트 브라우저를 사용하여 IBM Navigator for i에 대한 기능 실행
  • SQL 스크립트 및 SQL Performance Center 실행과 같은 데이터베이스 피처.

선택적인 Windows Application Package에는 다음이 포함됩니다.

  • ODBC, .Net 및 OLE DB를 사용하여 i용 Db2에 연결
  • IBM i 시스템 오브젝트에 액세스하기 위한 프로그래밍 툴킷
  • TLS/SSL 연결에 대한 지원
  • AFP 프린터 드라이버

선택적인 Linux Application Package에는 다음이 포함됩니다.

  • i용 Db2에 액세스하고 전체 64비트 ODBC 데이터 유형을 지원하는 ODBC 드라이버

3.1 요구사항(워크스테이션)

IBM i Access Client Solutions는 Linux, Mac 및 Windows의 여러 버전을 포함하여 Java 8.0 이상을 지원하는 대부분의 오퍼레이팅 시스템에서 실행됩니다.

권장사항:
사용자의 Java 버전을 최신 버전으로 유지하면 최신 수정 프로그램과 보안 패치 모두를 확보할 수 있습니다.

    java -version

다음 출력은 버전 8.0이 설치되었음을 나타냅니다.
    java version "1.8.0_65"

다음은 Java 제공자의 일부 웹 사이트입니다. 사용자의 플랫폼에 맞는 최신 버전의 Java에서 실행 중인지 확인하십시오.

    http://openjdk.java.net/install

5.0 설치

IBM i Access Client Solutions 패키지에서는 아주 간단한 단일 사용자 설치부터 고급 다중 사용자 설치까지 여러 설치 옵션을 제공합니다.

설치 스크립트는 설치 옵션에 따라 사용할 수 있는 Mac_Application, Linux_Application 및 Windows_Application 폴더(4.0 제품 컨텐츠 섹션 참조)에서 제공됩니다.

Mac, Linux 및 Windows에서의 단일 사용자 설치 또는 Windows PC에서의 다중 사용자 설치의 경우, 빠른 시작 안내서를 참조하십시오. 빠른 시작 안내서는 이러한 유형의 단순 설치를 수행하기 위해 제공된 설치 스크립트의 사용법에 대해 설명합니다.

여러 사용자에게 이를 배치할 계획인 관리자의 경우, IBM i Access Client Solutions: 손쉬운 사용자 정의 및 배치 기사를 읽으십시오. 기사에서는 관리자가 설치 스크립트에서 지원하는 /AdminConfig 옵션을 사용하여 배치하기 전에 다중 사용자용 설치를 미리 사용자 정의하는 방법에 대해 설명합니다.

자동 모드에서 설치를 자동화하려는 관리자의 경우, /AdminConfig 매개변수를 사용하여 설치를 미리 구성하십시오. 그런 다음 실제 설치 도중 /Q 매개변수를 사용하십시오. 예를 들면, 다음과 같습니다.

    Windows_Application\install_acs_64.js /AdminConfig
    Windows_Application\install_acs_64.js /Q

위 옵션과 함께 사용자가 원하는 위치에 zip 아카이브를 압축 해제하는 옵션도 있습니다. 이는 워크스테이션(PC, Mac 등)에 파일에 액세스할 수 있는 읽기 권한이 있는 위치입니다. 해당 위치에는 로컬 하드 디스크 드라이브, 리모트 네트워크(공유) 드라이브 또는 휴대용 매체(예: CD 또는 USB 플래시 드라이브)가 포함됩니다. zip 아카이브를 압축 해제하면 설치가 완료됩니다.

기술적 참고사항:
일부 아카이브 유틸리티는 저장된 모든 파일 속성을 보존하지 않습니다. 예를 들어, Mac과 Linux 플랫폼에서는 unzip 명령이 jar 명령보다 좋은 선택인 경우가 많습니다. 추가 정보는 6.0 파일 허용 절을 참조하십시오.

5.1 기존 설치 갱신

정기적으로 개선 및 수정 프로그램을 사용할 수 있습니다. 이러한 갱신사항은 전체 제품 설치로 제공됩니다. 이러한 갱신사항이 사용 가능한 경우, 기존 설치를 갱신해야 합니다.

설치 스크립트를 사용하여 설치한 기존 설치를 갱신하려면 새 버전의 제품 컨텐츠를 추출하십시오. 초기 설치에서와 같은 방식으로 이 새 버전에서 설치 스크립트를 호출하십시오. 그러면 사용자의 기존 구성을 변경하지 않고 제품 파일을 갱신합니다.

여러 사용자가 액세스하는 중앙 위치에서 제품 버전을 유지보수하려는 관리자의 경우, 기존 버전 위에 새 버전의 컨텐츠를 추출할 수 있습니다. AcsConfig.properties 파일을 사용자 정의한 경우, 기존 버전 위에 새 제품 버전의 컨텐츠를 추출하기 전에 AcsConfig.properties를 저장하려 합니다. 새 제품 버전의 컨텐츠를 추출하고 AcsConfig.properties를 복원한 후에는 사용자가 초기 설치와 같은 방식으로 설치 스크립트를 실행하여 설치를 갱신할 수 있습니다.

사용자가 선택한 위치에 zip 아카이브의 컨텐츠를 추출함으로써 설치 스크립트를 사용하지 않고 제품을 설치한 후 제품을 갱신하도록 선택하는 경우, 기존 버전 위에 새 제품 버전의 컨텐츠를 추출해야 합니다. 사용자 정의한 경우 AcsConfig.properties의 컨텐츠를 저장한 후 복원해야 함을 기억하십시오.

6.0 파일 허용

    ls -l <file>

파일 허용을 변경하려면 다음 명령을 사용할 수 있습니다.

    chmod <permission> <file>

7.2 제품 시작(스크립트 사용)

v1r1m5부터 Start_Scripts 디렉토리는 Sample_Scripts로 이름이 변경되어 문서 디렉토리에 있습니다.

선호하는 제품 시작 방법은 Start_Programs에서 플랫폼 특정 2진 파일을 사용하는 것입니다. 플랫폼 특정 2진 파일이 작동하지 않으면 Sample_Scripts에 있는 스크립트만 사용해야 합니다.

제공된 스크립트 중 하나를 사용하여 파일 시스템 브라우저(예: Windows Explorer, Mac OS X Finder 등)에서 제품을 시작하려면, 사용자 오퍼레이팅 시스템과 호환되는 Sample_Scripts 서브디렉토리에서 스크립트를 찾으십시오.

대부분의 비Windows 기반 오퍼레이팅 시스템에서는 디폴트로 perl을 사용할 수 있습니다. Sample_Scripts/Linux_Mac_Other 디렉토리에는 perl을 사용할 수 있는 플랫폼에서 제품을 시작할 때 사용할 수 있는 perl 스크립트(서로 다른 3개의 파일 확장자가 있음)가 있습니다. 오퍼레이팅 시스템이 perl 스크립트로 인식할 파일 확장자가 있는 파일을 선택하십시오.

Windows 기반 오퍼레이팅 시스템에서는 디폴트로 JScript를 사용할 수 있습니다. Sample_Script/Windows 디렉토리에는 Windows 오퍼레이팅 시스템에서 제품을 시작하는 데 사용할 수 있는 JScript가 들어 있습니다.

플랫폼 특정 방법을 사용하여 파일 시스템을 찾으려면(예: Windows Explorer, Mac OS X Finder 등), 오퍼레이팅 시스템이 인식하는 스크립트를 찾으십시오. 그 다음, 해당 스크립트를 두 번 클릭하여 제품을 시작하십시오. 또한 명령 프롬트, 단말기 또는 쉘에서 해당 스크립트로 제품을 시작할 수도 있습니다.

7.3 제품 시작(명령행 사용)

명령(명령 프롬트, 단말기, 쉘 등)을 입력할 수 있는 위치에서 명령행으로 제품을 시작할 수도 있습니다.

    java -Xmx1024m -jar <path>/acsbundle.jar
여기서 <path>는 제품의 실행 가능 jar 파일의 위치입니다.

예를 들면, 다음과 같습니다.
    java -Xmx1024m -jar V:/some_location/acsbundle.jar
또는 
    <java_path>java -jar V:/some_location/acsbundle.jar
여기서 <java_path>는 JDK 6.0 이상에서 Java 명령의 위치입니다.

Java 명령에 대한 전체 경로를 판별하려면 7.1.2 Java 홈 경로 찾기 절을 참조하십시오.

명령행에서 프로그램 또는 스크립트를 사용할 수도 있습니다. 예를 들면, 다음과 같습니다.

    /Product/Location/Start_Programs/Linux_x86-64/acslaunch_linux-64
    /Product/Location/Start_Programs/Mac_i386-32_x86-64/acslaunch_mac
    C:\Product\Location\Start_Programs\Windows_x86-64\acslaunch_win-64.exe

기술적 참고사항:
대부분의 플랫폼에서 JVM(Java Virtual Machine) 힙 공간의 디폴트 값은 IBM i Access Client Solutions 제품 내에서 복수의 기능을 이용하기에 너무 작은 최대 크기입니다. 1GB의 최대 힙 크기(-Xmx1024m)는 권장되는 최소 크기입니다. 크기를 1GB보다 작게 지정하거나 디폴트 힙 크기를 사용하면 OutOfMemoryException이 발생할 수 있습니다.

8.0 구성

사용하거나 관리하려는 각 IBM i 시스템에 대한 시스템 구성을 추가하십시오. 시스템 구성을 추가하려면 관리 태스크에서 시스템 구성을 선택하십시오. 그 다음, 새로 작성을 선택하십시오. 일반 탭에 시스템명을 입력하십시오. 시작하기 위해 일반 태스크를 수행하려면 시스템명만 입력하면 됩니다.

완료한 후, 이 시스템에 입력한 정보를 저장하려면 확인을 클릭하고, 구성에 추가할 추가 시스템이 있는 경우에는 저장/새로 작성을 선택하십시오.

구성에 새 시스템을 추가하거나 언제든 일반, 연결 또는 콘솔 탭을 사용하여 기존 구성을 갱신할 수 있습니다.

콘솔 태스크의 경우, 추가 구성이 필요합니다. 콘솔 구성은 일반 탭에서 입력한 시스템명과 자동으로 연관됩니다. 시스템의 콘솔 구성을 입력하려면 관리 태스크에서 시스템 구성을 선택하십시오. 새로 작성 또는 편집을 선택하십시오. 그 다음 콘솔 탭을 선택하십시오. 5250 콘솔 태스크에는 구성된 LAN 콘솔 또는 구성된 HMC 콘솔이 필요합니다. 구성된 LAN 또는 HMC 콘솔이 없는 경우 9.9 IBM i에 콘솔 연결 설정 절을 참조하십시오.

하드웨어 관리 인터페이스 태스크에는 구성된 하드웨어 관리 인터페이스가 필요합니다. 최대 두 개의 하드웨어 관리 인터페이스 구성을 입력할 수 있습니다.

완료한 후, 시스템 구성 패널에서 닫기를 선택하십시오.

기본 IBM i Access Client Solutions 패널에서 시스템 드롭 다운 상자를 사용하여 시스템을 선택하십시오. 모든 콘솔 태스크에서는 선택한 시스템(일반 탭에서 입력)을 콘솔 구성(콘솔 탭에서 입력)과 자동으로 연관시킵니다.

이제 선택한 시스템의 태스크를 선택할 수 있습니다. 콘솔 구성 탭에 입력한 해당 정보가 없는 콘솔 태스크를 선택하는 경우, 오류 메세지가 표시됩니다.

9.2 파일 연관

IBM i Access Client Solutions에서 생성된 일부 구성 파일은 처음이자 유일한 매개변수로 제공되는 경우 명령행을 통해 지원됩니다. 특정 파일 확장자를 가진 이러한 파일을 첫 번째 매개변수로 제공하는 경우, IBM i Access Client Solutions가 호출할 기능과 파일을 연관시키고 해당 기능에 대한 입력으로 파일을 제공합니다.

다음 확장자가 파일 연관을 지원합니다. 

    .dttx   - 데이터 전송 업로드 요청
    .dtfx   - 데이터 전송 다운로드 요청
    .hod    - 5250 에뮬레이터 세션 프로파일
    .bchx   - 복수 세션 에뮬레이션 프로파일
    .ws     - 5250 에뮬레이션 세션 프로파일(퍼스널 통신)
    .bch    - 복수 세션 에뮬레이션 프로파일(퍼스널 통신)
명령행 예: 
    acslaunch_xxx dt_download_file.dtfx - 저장된 다운로드 조작 실행
    acslaunch_xxx dt_upload_file.dttx   - 저장된 업로드 조작 실행
    acslaunch_xxx system_lp13ut20.hod   - 시스템에 대한 5250 세션 시작
여기서 acslaunch_xxx는 제품 시작에 사용되는 명령행 구문입니다. 7.3 제품 시작(명령행 사용) 절을 참조하십시오.

지원되는 명령행 파일 연관을 작동하면 사용자가 오퍼레이팅 시스템(OS) 특정 파일 연관을 수동으로 설정할 수 있습니다. 파일 연관은 플랫폼과 관련되지 않으므로, 필요한 단계는 OS에 따라 달라집니다.

파일(위에서 지원되는 파일 유형 중 하나)을 두 번 클릭하여 지정된 기능을 시작하는 기능이 있으므로 OS에 대한 파일 연관 설정을 고려할 수 있습니다.

다음 절에서는 일부 오퍼레이팅 시스템에 대한 파일 연관을 설정하는 예를 제공합니다.

9.2.2.1 파일 연관(Linux)

파일 연관 설정에 필요한 단계는 사용 중인 Linux 분배 및 데스크탑 환경에 따라 다릅니다. 일반적으로 필요한 단계는 Windows에 사용되는 위의 단계와 유사합니다.

  1. 지원되는 확장자(예: .hod, .bchx, .dtfx 또는 .dttx)가 있는 연관시킬 파일을 찾으십시오.

  3. 파일 및/또는 확장자를 해당 IBM i Access Client Solutions 2진 파일 또는 실행 스크립트와 연관시키십시오.

9.3 구성 위치 변경

기본적으로 각 사용자의 구성에는 고유한 위치가 있습니다. 구성 위치는 다음 등록정보를 설정하여 변경할 수 있습니다.
    com.ibm.iaccess.AcsBaseDirectory
이 등록정보는 AcsConfig.properties 파일에 있습니다.

AcsConfig.properties 파일은 제품이 제공될 때 두 개의 위치에 있습니다. 해당 파일은 acsbundle.jar 파일 내에 포함되어 있습니다. 편의상 파일은 제품 zip 아카이브 파일에도 제공되며 zip 아카이브가 압축 해제될 때 acsbundle.jar 파일과 동일한 디렉토리에 있습니다.

시작 시, 제품은 발견한 첫 번째 AcsConfig.properties 파일만 사용합니다. 먼저 acsbundle.jar 파일이 있는 디렉토리를 검사합니다. AcsConfig.properties가 acsbundle.jar 파일과 동일한 디렉토리에 없는 경우 acsbundle.jar 파일에 있는 AcsConfig.properties 파일을 사용합니다.

acsbundle.jar의 AcsConfig.properties를 사용자 정의 구성 경로로 갱신하도록 선택할 수 있습니다. 선택하는 경우 acsbundle.jar가 있는 디렉토리에 AcsConfig.properties 파일이 없거나 대신 해당 디렉토리가 사용되는지 확인하십시오. 이를 통해 acsbundle.jar 파일이 포함된 구성 위치를 분배할 수 있으며 구성 위치를 대체할 수 있게 됩니다.

구성 경로를 정의할 때 사용될 수 있는 특수 키워드가 제공됩니다. 지정된 경로에 키워드가 사용되는 경우 키워드는 정의한 텍스트 또는 경로로 대체됩니다. 구성 경로에는 하나의 키워드만 사용될 수 있습니다. 특수 키워드 및 해당 의미는 다음과 같습니다.

	{USER} - 현재 사용자 ID. 이 키워드는 경로의 임의 위치가 될 수 있습니다.

다음 키워드는 지정된 경로 시작 부분에만 있을 수 있습니다. 

	{PRODUCTDIR} - acsbundle.jar와 동일한 디렉토리에 대한 경로
	{TEMPDIR}    - 플랫폼 특정 임시 디렉토리의 경로
	{ROOT}       - 파일 시스템의 루트에 대한 경로
	{HOME}       - 사용자의 홈 디렉토리에 대한 경로
	{DEFAULT}    - 제품에서 일반적으로 사용하는 디폴트 경로

기술적 참고사항:
여러 사용자가 구성을 공유하지 않는 것이 좋습니다. 예를 들어 X가 공유 네트워크 드라이브인 경우, 다음 설정으로 인해 예측 불가능한 결과가 발생할 수 있습니다. 

    com.ibm.iaccess.AcsBaseDirectory=X:/Shared_Network_drive/config_directory

이 구성 경로를 여러 사용자가 공유하면 여러 가지 문제가 있습니다.

  1. 여러 사용자가 동시에 구성을 사용하면 예측 불가능한 결과가 발생합니다.
  2. 이 예에서는 각 사용자가 X 드라이브를 동일한 위치에 맵핑한 것으로 가정합니다.
  3. 구성에서 네트워크 드라이브를 사용하는 것은 Linux 또는 Mac 클라이언트에서 적용되지 않습니다. 아래 예와 같이 제공된 키워드를 사용하면 Windows, Linux 및 Mac에서 모두 작동합니다.

복수 사용자 간에 구성을 공유할 때는 다른 사용자와 충돌을 피하기 위해 {USER} 키워드를 사용해야 합니다. 이는 현재 사용자의 사용자 ID로 대체됩니다.

구성 경로를 설정할 때는 디렉토리 분리자로 백슬래시('\') 대신 슬래시('/')를 사용하십시오. 이렇게 하면 Windows를 포함한 모든 오퍼레이팅 시스템에서 작동합니다.

아래 내용은 몇 가지 권장 샘플 구성입니다.

예 1 - 현재 사용자의 로컬 구성(디폴트): 

    com.ibm.iaccess.AcsBaseDirectory=
AcsBaseDirectory가 설정되지 않으면 구성은 사용자의 플랫폼별 경로를 디폴트로 사용합니다. 이는 IBM i Access Client Solutions의 디폴트 설정입니다.

예 2 - 각 사용자에게 고유한 리모트(또는 로컬) 구성: 

    com.ibm.iaccess.AcsBaseDirectory={ROOT}/config_directory/{USER}/
구성은 {ROOT} 위치를 기반으로 하는 리모트 또는 로컬입니다.

예 3 - 각 사용자에게 고유한 리모트(또는 로컬) 구성: 

    com.ibm.iaccess.AcsBaseDirectory={PRODUCTDIR}/config_directory/{USER}/
구성은 {PRODUCTDIR} 위치를 기반으로 하는 리모트 또는 로컬입니다.

예 4 - 휴대용 매체(예: USB 드라이브)에서의 로컬 구성: 

    com.ibm.iaccess.AcsBaseDirectory={PRODUCTDIR}/config_directory
휴대용 매체에 대한 경로는 사용 중인 시스템에 따라 달라지므로 이렇게 설정하면 제품 파일에 따라 구성이 달라질 수 있습니다. 이 예에서는 USB 드라이브에 대한 구성을 사용자 ID에 관계 없이 USB 드라이브의 사용자가 사용할 수 있도록 의도적으로 {USER} 키워드를 지정하지 않았습니다.

9.4 기타 배치 옵션

다음은 고려해야 하는 기타 배치 옵션입니다.

  • 여러 사용자가 액세스할 수 있는 단일 위치에 제품을 배치합니다.
  • 웹 서버가 있는 경우 Web Start 사용을 고려할 수 있습니다. http://java.com/en/download/faq/java_webstart.xml을 참조하십시오.
  • 구성 설정값을 자동으로 가져옵니다. 9.4.1 절을 참조하십시오.
  • 기본적으로 IBM i에 배치합니다. 9.4.2 섹션을 참조하십시오.

9.4.1 구성 설정값 자동으로 가져오기

AcsConfig.properties 파일에서 다음 등록정보를 사용하여 자동으로 새 사용자의 구성을 설정하거나 기존 사용자의 구성을 갱신할 수 있습니다.

   com.ibm.iaccess.autoimport
   com.ibm.iaccess.autoimport.version

다음은 필수 단계입니다.

  1. 하나 이상의 사용자에게 전파하려는 구성을 작성하십시오.
  2. 기본 GUI에서 파일->구성 내보내기를 사용하여 구성을 파일로 내보내거나 9.1.1 백업 절의 명령행 옵션을 사용하십시오.
  3. 원하는 위치로 구성 파일을 이동하십시오. 여러 사용 가능한 옵션이 나중에 설명됩니다.
  4. AcsConfig.properties에서 com.ibm.iaccess.autoimport 등록정보를 구성 파일의 경로로 설정하십시오.
  5. 구성 파일 버전을 나타내는 정수 값으로 com.ibm.iaccess.autoimport.version을 설정하십시오.

작동 방식은 다음과 같습니다.
com.ibm.iaccess.autoimport.version 등록정보의 정수 값이 가져온 최신 값과 일치하지 않으면 com.ibm.iaccess.autoimport 등록정보에서 참조하는 저장된 구성을 자동으로 가져옵니다. 초기 구성을 설정하고 기존 구성을 갱신하는 방법을 제공하는 것 이외에 구성을 백 레벨로 조정하는 방법도 제공합니다. 버전 등록정보의 정수 값과 가져온 최근 값이 불일치하는 경우 언제라도 구성이 갱신됩니다. 그러나 가져온 구성이 사용자 구성의 기존 시스템에 대한 구성을 변경할 수 있지만 사용자 구성에서 시스템이 삭제되지는 않습니다.

com.ibm.iaccess.autoimport 등록정보에 대한 경로는 9.3 절에 정의된 절대 경로 URL 또는 키워드로 지정될 수 있습니다. 예를 들면, 다음과 같습니다. 

    com.ibm.iaccess.autoimport=C:/acs_bak.zip

    com.ibm.iaccess.autoimport=file///C:/acs_bak.zip

    com.ibm.iaccess.autoimport=http://your.company.com/path/file/acs_bak.zip

    com.ibm.iaccess.autoimport=ftp://your.company.com/path/file/acs_bak.zip

    com.ibm.iaccess.autoimport={PRODUCTDIR}/acs_bak.zip

구성 파일이 acsbundle.jar에서 또는 acsbundle.jar와 동일한 디렉토리에서 분배되도록 하여 추가적인 유연성이 제공됩니다. 두 경우 모두 선행 경로가 없는 파일 이름으로 com.ibm.iaccess.autoimport를 설정하십시오.

    com.ibm.iaccess.autoimport=acs_bak.zip

com.ibm.iaccess.autoimport.version에 대해서는 특수 값 *가 허용됩니다. 

    com.ibm.iaccess.autoimport.version=*

이전 버전에 관계 없이 항상 구성을 가져옵니다.

9.4.2 기본 IBM i 배치

GUI가 필요하지 않은 명령행 플러그인 일부는 IBM i에서 기본적으로 사용할 수 있습니다. 예를 들어 PC에 데이터를 다운로드하지 않고 IBM i에서는 데이터 전송을 사용하여 사용자 데이터베이스에서 스프레드시트 파일 유형으로 데이터를 직접 추출할 수 있습니다.

이를 위해 제품 zip 아카이브를 IBM i IFS(Integrated File System) 위치로 추출할 수 있습니다. 예를 들어 제품 zip 아카이브를 다음 위치로 추출하고, 

 /home/AccessClientSolutions
다음 위치에 이전에 저장된 데이터 전송 다운로드 요청이 있는 경우, 
 /some_path/qcustcdt.dtfx
IBM i에서 다음 명령을 실행할 수 있습니다. 
QSH CMD('java -jar /home/AccessClientSolutions/acsbundle.jar /plugin=dtbatch /some_path/qcustcdt.dtfx')

  debugger          sysdbg
시스템 관리자는 AcsConfig.properties 파일을 갱신하여 특정 기능에 대한 액세스를 제한할 수 있습니다. 액세스를 제한하면 해당 기능이 기본 GUI에서 제거되며 명령행에서 이를 작동 불가능하게 됩니다.

기능 제한 방법에 대한 한 가지 예로 다음 행을 AcsConfig.properties 파일에 추가하십시오.

    com.ibm.iaccess.ExcludeComps=OPCONSOLE,HWCONSOLE,L1CPLUGIN
시스템 관리자가 acsbundle.jar 내부의 AcsConfig.properties 파일을 사용자에게 배치하기 전에 갱신하려는 경우 이를 수행하는 방법에 대한 예는 다음과 같습니다.
    jar uvf acsbundle.jar AcsConfig.properties
IBM i Access Client Solutions가 사용할 AcsConfig.properties 파일을 결정하는 방법에 대한 추가 정보는 9.3 구성 위치 변경 절을 참조하십시오.

9.8 데이터 전송

9.8.1 Excel 및 Calc 스프레드시트에 대한 데이터 전송 지원

데이터 전송은 파일로의 다운로드를 지원할 뿐만 아니라 Microsoft의 Excel 스프레드시트 또는 OpenOffice의 Calc 스프레드시트에 있는 활성 스프레드시트로의 다운로드도 지원합니다. 활성 스프레드시트로 다운로드하기 위해 기본 데이터 전송 GUI 패널에는 출력 장치를 선택할 수 있는 옵션이 제공됩니다. 디폴트 출력 장치는 파일입니다. 플랫폼이 Excel 및/또는 Calc 스프레드시트와의 상호작용을 지원하는 경우 활성 Excel 스프레드시트 및 활성 Calc 스프레드시트에 대한 추가 옵션을 출력 장치 드롭 다운 상자에서 선택할 수 있습니다.

제한사항:

  • Windows에서는 최소 OpenOffice 3.4.1 이상의 버전이 필요합니다.
  • Linux와 Mac에서는 최소 OpenOffice 4.2.1 이상의 버전이 필요하며 디폴트 위치에 설치되어 있어야 합니다.
  • Linux 또는 Mac에서 실행되는 IBM i Access Client Solutions는 OpenOffice와의 상호 작용만 지원합니다.
    Windows에서 실행되는 IBM i Access Client Solutions는 Microsoft Excel 또는 OpenOffice Calc와의 상호 작용을 지원합니다.
  • OpenOffice의 비트 수가 IBM i Access Client Solutions의 비트 수와 일치해야 합니다.
    설명: OpenOffice는 32비트 및 64비트 버전으로 제공됩니다. 두 버전 모두 64비트 PC에서 실행할 수 있습니다. OpenOffice 스프레드시트와 상호 작용할 때 OpenOffice 및 IBM i Access Client Solutions의 비트 수가 동일해야 합니다. 그렇지 않으면 데이터 전송이 스프레드시트와 상호 작용할 수 없습니다. IBM i Access Client Solutions의 비트 수는 애플리케이션이 실행 중인 JVM(Java Virtual Machine)에 의해 판별됩니다. OpenOffice와 IBM i Access Client Solutions를 실행하는 JVM 사이에서 비트 수가 일치하지 않은데 출력 장치에 대한 활성 스프레드시트를 선택하는 경우 "활성 스프레드시트를 찾을 수 없음" 오류가 표시됩니다. 64비트 PC에서 실행하는 경우 사용자 플랫폼에 적합한 32비트 또는 64비트 2진 실행기를 사용하여 IBM i Access Client Solutions를 시작하게 됩니다. 활성 스프레드시트 지원이 작동하게 하려면 사용 중인 OpenOffice 애플리케이션의 비트 수와 일치하는 플랫폼 고유의 2진을 사용하여 IBM i Access Client Solutions를 시작하십시오.
    Mac은 64비트 JVM만을 지원하므로 Mac에서 활동 스프레드시트 지원을 사용하려면 64비트 버전의 OpenOffice를 설치해야 합니다.

9.8.2 문자 절단 및 숫자 넘침에 대한 데이터 전송 지원

데이터 전송 업로드 요청 도중 문자 또는 숫자 필드가 정의된 필드 크기를 초과하면 업로드 요청이 종료됩니다.

문자 필드를 끝에서부터 절단하려면 AcsConfig.properties 파일에서 다음 특성을 설정하십시오.
    com.ibm.iaccess.dataxfer.jdbc.AllowCharacterTruncation=true

제공된 숫자 필드가 정의된 경계를 초과하면 숫자 필드를 최대 양의 값 또는 음의 값으로 설정할 수 있도록 하려면 AcsConfig.properties 파일에서 다음 특성을 설정하십시오.
    com.ibm.iaccess.dataxfer.jdbc.AllowNumericOverflow=true

이러한 특성이 true로 설정되면 절단 또는 넘침이 발생했음을 표시하지 않고도 업로드 요청이 계속됩니다. 문자 필드는 끝에서 절단됩니다. 숫자 필드는 넘침의 경우 최대값으로 설정되고 언더플로의 경우 최소값으로 설정됩니다.

또는 다음과 같은 기타 Java 특성과 같이 위 특성을 명령행에서 설정할 수 있습니다.
    -Dcom.ibm.iaccess.dataxfer.jdbc.AllowCharacterTruncation=true
    -Dcom.ibm.iaccess.dataxfer.jdbc.AllowNumericOverflow=true

9.8.3 데이터 전송 시트 이름

다운로드 중 IBM i 소스 라이브러리의 이름을 기반으로 시트 이름이 생성되고, 파일에는 ">Sheet#"가 추가됩니다. 여기서 #은 해당 시트 번호로 대체됩니다. 예를 들면, 다음과 같습니다.
    qiws.qcustcdt>Sheet1

시트 이름의 라이브러리 및 파일 부분을 대체하기 위해 다음 특성을 지정할 수 있습니다.
    com.ibm.iaccess.dataxfer.SheetId=your_string
다음이 생성됩니다.
    your_string>Sheet1

9.9 IBM i에 대한 콘솔 연결 설정

IBM i에 관리 기능을 수행하려면 5250 콘솔이 필요합니다. IBM i Access Client Solutions는 LAN 및 HMC 콘솔 구성을 모두 지원합니다.

IBM i의 서비스 호스트명이나 서비스 IP 주소 또는 HMC 콘솔의 호스트명이나 IP 주소를 알고 있는 경우, 다음 단계에 따라 IBM i Access Client Solutions 내에서 콘솔 정보를 구성할 수 있습니다.

  1. 사용자 PC에서 IBM i Access Client Solutions를 시작하십시오.
  2. 기본 패널에서 시스템 구성을 클릭하십시오.
  3. 새로 작성 버튼을 클릭하여 새 구성을 입력하거나 편집 버튼을 클릭하여 기존 구성을 갱신하십시오.
  4. 콘솔 탭을 선택하십시오.
  5. 콘솔 유형에 대해 적합한 정보를 입력하십시오.

콘솔 구성이 아직 없는 IBM i의 경우(즉, 사용자 비즈니스에 바로 도입된 새 시스템), 콘솔 연결을 위한 IP 주소가 IPL 중에 169.254.62.0 - 169.254.62.63 범위 내에서 자동으로 할당됩니다. 이런 경우 다음 단계는 IBM i Access Client Solutions를 사용하여 콘솔 연결을 설정하는 데 도움이 됩니다.

  1. 사용자 PC의 무선 연결을 사용 안함으로 설정하십시오. 다음 단계에는 유선 이더젯 포트가 필요합니다.
  2. 사용자 PC와 IBM i T1 포트 사이에 이더넷 케이블을 연결하십시오.
  3. 사용자 PC에서 IBM i Access Client Solutions를 시작하십시오.
  4. 기본 패널에서 시스템 구성을 클릭하십시오.
  5. 시스템 구성에서 콘솔 찾기... 버튼을 클릭하십시오.
  6. IBM i 위치 지정자 패널에서 탐색 버튼을 클릭하십시오.
  7. 자동 IP 주소가 할당될 정도로 IBM i의 IPL 시퀀스가 많이 진행된 경우 IP 주소가 탐색 패널에 표시되어야 합니다.
  8. 표시된 IP 주소를 클릭하여 선택한 다음, 콘솔 버튼을 클릭하여 5250 콘솔에 연결하십시오.
  9. 서비스 툴 신임 정보를 묻는 패널이 표시됩니다.
  10. 유효한 신임 정보를 입력하면 5250 콘솔이 표시됩니다.
  11. 콘솔을 사용하여 초기 설치 및 설정 태스크를 처리할 수 있습니다.

IBM i 시스템을 나머지 네트워크 인프라에 추가할 준비가 되면, 서비스 툴 LAN 어댑터의 구성 방법에 대한 추가 정보가 IBM InfoCenter에서 제공됩니다. 

http://pic.dhe.ibm.com/infocenter/iseries/v7r1m0/topic/rzamh/rzamhsrvtoolsrvr4dstdst.htm
"DST에 대한 서비스 툴 서버 구성" 절을 참조하십시오.

이미 LAN 콘솔 구성이 있고 이미 네트워크 인프라에 있는 IBM i 시스템의 경우, 다음 단계를 사용하여 기존 콘솔 구성을 찾을 수 있습니다.

  1. 사용자 PC에서 IBM i Access Client Solutions를 시작하십시오.
  2. 기본 패널에서 시스템 구성을 클릭하십시오.
  3. 시스템 구성에서 콘솔 찾기... 버튼을 클릭하십시오.
  4. IBM i 위치 지정자 패널에서 탐색 범위에 있는 IP 주소로 근접 필드를 갱신하십시오.
  5. 탐색 버튼을 클릭하여 지정된 범위의 LAN 콘솔을 찾으십시오.
  6. 표시된 IP 주소를 클릭하여 선택한 다음, 콘솔 버튼을 클릭하여 5250 콘솔에 연결하십시오.
  7. 서비스 툴 신임 정보를 묻는 패널이 표시됩니다.
  8. 유효한 신임 정보를 입력하면 5250 콘솔이 표시됩니다.

9.10 추가 폰트

5250 에뮬레이션을 위한 추가 폰트가 폰트 디렉토리에 추가될 수 있습니다.

추가 폰트의 대체 및/또는 다중 위치를 제공하려면 AcsConfig.properties 파일에 있는 com.ibm.iaccess.Fonts 등록정보를 설정하십시오.

기본 Fonts 디렉토리는 com.ibm.iaccess.Fonts 등록정보를 다음으로 설정하여 대체할 수 있습니다.

  1. <path> - 폰트가 들어 있는 디렉토리
  2. 특정 폰트 파일의 <path>/<filename>(예: .ttf)
  3. 폰트 파일의 URL
  4. 세미콜론으로 구분된 위 내용의 리스트

주:
<path>는 acsbundle.jar 위치에서 시작하는 절대 경로 또는 상대 경로일 수 있습니다. 5250 에뮬레이션에서는 모노스페이스 폰트만 인식합니다.

예 1(디폴트): 

     com.ibm.iaccess.Fonts=Fonts
예 2: 
     com.ibm.iaccess.Fonts=/Users/All/Fonts;Fonts

9.11 netrc 파일에서 신임 정보 사용

.netrc 파일(Linux 및 Mac) 또는 a _netrc 파일(Windows)을 사용할 수 있도록 하려면 IBM i Access Client Solutions 기본 GUI로 이동하여 메뉴 바에서 다음을 선택하십시오.

    일반 탭
    상자 선택: netrc 파일에서 로그인 정보 읽기
    적용 버튼 클릭
    IBM i Access Client Solutions 재시작

netrc 파일의 표준 형식은 다음과 같습니다. 

     machine <system> login <user-id> password <password>

netrc 파일은 사용자의 홈 디렉토리에 저장되어야 하고 시스템명과 사용자 ID는 시스템 구성과 일치해야 합니다. IBM i Access Client Solutions에서는 다음을 수행하십시오.
    시스템 구성 선택
    시스템 선택 후 편집
    연결 탭
    디폴트 사용자 이름을 사용하여 각 시스템에 대해 한 번씩 프롬트 선택
    사용자 ID 채우기: 디폴트 사용자명
    확인 선택

시스템명과 디폴트 사용자명이 netrc 파일의 컨텐츠와 일치하면, 사용자에게 프롬트를 표시하지 않고 netrc 파일의 암호를 사용하여 시스템에 연결합니다.

9.12 IFS(Integrated File System)

9.12.1 IFS, QFileSvr.400 및 보안

IFS(Integrated File System) 지원에서는 QFileSvr.400 파일 시스템을 사용하여 하나의 IBM i 파티션에서 다른 파티션으로 오브젝트를 복사하거나 전송합니다. 이는 두 파티션 모두에서 사용자 프로파일과 암호가 일치하고 파티션의 암호 레벨 시스템 값(QPWDLVL)이 같아야 함을 의미합니다. Kerberos 인증을 사용하는 경우, 두 파티션 모두에는 네트워크 인증 서비스와 EIM(Enterprise Identity Mapping)이 구성되어 있어야 합니다. 자세한 내용은 IBM i Knowledge center의 "QFileSvr.400 파일 시스템에서 보안 및 오브젝트 권한" 주제를 참조하십시오.

QFileSvr.400 파일 시스템에서 작성한 다른 IBM i 파티션으로의 연결은 안전하지 않습니다. 하나의 IBM i 파티션에서 다른 파티션으로 오브젝트를 복사하거나 전송할 때 보안 연결이 필요한 경우, 파티션에 대한 구성은 SSL을 사용하도록 설정되어야 하고, 시스템 구성에서는 해당 파티션에 대해 "연결에 SSL 사용" 옵션이 선택되어야 하며, 다음 특성이 AcsConfig.properties 파일에서 설정되어 있어야 합니다.
    com.ibm.iaccess.ifs.transferMechanism=ACS
또는 다음과 같은 다른 Java 특성과 같이 명령행에서 특성을 설정할 수 있습니다.
    -Dcom.ibm.iaccess.ifs.transferMechanism=ACS

9.12.2 IFS 권한

통합 파일 시스템 지원에서는 다음 CL 명령을 사용하기 위한 권한이 필요합니다.

사용자가 이러한 CL 명령을 사용할 수 없으면, 복사, 붙여넣기, 송신, 삭제 및 폴더 새로 작성 조치가 실패합니다.

9.12.3 IFS 제한사항

통합 파일 시스템 지원은 경로가 /QFileSvr.400, /QNTC 또는 /QOPT로 시작하는 오브젝트에 대해 조치를 허용하지 않습니다. 현재 디렉토리 경로 또는 선택한 오브젝트 경로가 /QFileSvr.400, /QNTC 또는 /QOPT로 시작하는 경우, 조치 메뉴에 있는 모든 항목은 사용할 수 없고 문맥 메뉴는 표시되지 않습니다.

9.12.4 IFS 및 독립 ASP

통합 파일 시스템 지원에서는 소스 오브젝트 또는 대상 경로가 독립 ASP에 있는 경우 복사, 붙여넣기 또는 송신 조치를 허용하지 않습니다.

9.13 SSH(Secure Shell) 단말기

SSH(Secure Shell)가 디폴트로 있거나 설치된 플랫폼의 경우 SSH 단말기 피처가 IBM i Access Client Solutions 기본 GUI에 표시됩니다. SSH 단말기는 IBM i 통합 파일 시스템에 단말기 에뮬레이터를 실행합니다.

대부분의 Mac, Linux 및 기타 UNIX 파생 시스템에는 디폴트로 SSH(Secure Shell)가 있습니다. 이러한 플랫폼에서는 OpenSSH 클라이언트가 있고 /usr/bin/ssh에 있다고 가정합니다. Windows 사용자는 Windows에 사용할 수 있는 SSH(Secure Shell) 중 하나를 설치해야 합니다.

IBM i Access Client Solutions에서 지원되는 SSH(Secure Shell):

         Linux:       Xterm, XterminalEmulator, MateTerminal, Terminator, GnomeTerminal, Konsole, Xfce4
         Mac:         Terminal, iTerm2
         Windows: cygwin, PuTTY, bash
                          cygwin을 사용하려면 openssh 패키지가 설치되어 있어야 합니다.
                          bash는 Windows용 Linux 서브시스템의 일부로 사용할 수 있습니다.

복수의 SSH 애플리케이션이 설치된 경우 com.ibm.iaccess.PreferredSSHClient 특성을 사용하여 선호하는 SSH 클라이언트를 설정할 수 있습니다.

Linux 예:
         com.ibm.iaccess.PreferredSSHClient=XterminalEmulator
         com.ibm.iaccess.PreferredSSHClient=MateTerminal
         com.ibm.iaccess.PreferredSSHClient=Terminator
         com.ibm.iaccess.PreferredSSHClient=GnomeTerminal
         com.ibm.iaccess.PreferredSSHClient=Konsole
         com.ibm.iaccess.PreferredSSHClient=Xfce4

Mac 예:
         com.ibm.iaccess.PreferredSSHClient=iTerm2

Windows 예:
         com.ibm.iaccess.PreferredSSHClient=Cygwin
         com.ibm.iaccess.PreferredSSHClient=Putty
         com.ibm.iaccess.PreferredSSHClient=Bash

SSH 클라이언트가 표준 위치에 설치되지 않은 경우 com.ibm.iaccess.PreferredSSHClient 특성을 사용하여 설치된 SSH 클라이언트에 대한 완전한 경로를 제공할 수 있습니다.

         com.ibm.iaccess.PreferredSSHClient=C:\PuTTY\putty.exe

완전한 경로가 제공되면 필요한 모든 인수를 제공하도록 com.ibm.iaccess.SSHClientOpt를 설정해야 할 수도 있습니다. 예를 들어 X11 전달을 사용하지 않는 경우 경고를 작동 불가능하게 하려면 다음을 수행하십시오.

         com.ibm.iaccess.SSHClientOpts=-x

9.13.1 SSH 단말기 요구사항

IBM i에서 i용 5733-SC1 IBM Portable Utilities를 설치하십시오.
다음 명령을 사용하여 SSH(Secure Shell) 디먼을 시작하십시오.
         STRTCPSVR *SSHD

실행된 SSH 클라이언트 내에서 연결 오류를 수신하거나(예: "연결이 거부됨") 팝업 창이 표시되지만 즉시 없어지는 경우 위의 명령을 사용하여 SSH 디먼을 시작해야 할 수 있습니다.

Q21   패널에서 도움말 아이콘을 선택하여도 텍스트가 표시되지 않습니다.
A21-1 도움말 텍스트는 일반적으로 데스크탑 브라우저에 표시됩니다. 일부의 경우

      AcsConfig.properties 파일에서 다음 등록정보를 설정하십시오.      com.ibm.iaccess.javaAwtDesktopAllowed=false
      또한, 다음과 같은 기타 java 등록정보와 같이 명령행을 통해
      등록정보를 설정할 수도 있습니다.      -Dcom.ibm.iaccess.javaAwtDesktopAllowed=false
