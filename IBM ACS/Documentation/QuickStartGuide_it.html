1.0 Introduzione

Questa guida fornisce istruzioni su come distribuire il prodotto su un'unità disco locale per un singolo utente.

Ci sono diversi altri modi in cui questo prodotto può essere distribuito, tra cui:

  • un'ubicazione condivisa a cui possono accedere più utenti.
    Consulatare IBM i Access Client Solutions: Customization and deployment made easy
  • decomprimere l'archivio zip in un'ubicazione desiderata incluso un supporto portatile (ad es. unità CD, DVD, USB flash) o qualsiasi percorso sull'unità disco locale. Consultare la sezione Installazione del documento Introduzione situato nella cartella Documentazione
  • Consultare Ulteriori opzioni di distribuzione in Introduzione per opzioni come Importazione automatica delle impostazioni di configurazione per la propagazione della stessa configurazione a più utenti e Distribuzioni IBM i native per l'utilizzo dei moduli aggiuntivi di riga comandi direttamente su IBM i.

Per distribuire il prodotto sull'unità disco locale per un singolo utente, consultare le altre sezioni del documento.

2.0 Installazione per Windows

Individuare l'archivio zip o il CD del prodotto.

Se si utilizza l'archivio zip del prodotto, estrarne il contenuto.
Suggerimento: è possibile utilizzare Windows Explorer per selezionare l'archivio zip, quindi fare clic con il tasto destro del mouse e selezionare "Estrai tutto...".

Utilizzando Windows Explorer, individuare la cartella Windows_Application del prodotto.

Per installare la versione a 32 bit per l'utente corrente, fare doppio clic su install_acs_32.

Per installare la versione a 64 bit per l'utente corrente, fare doppio clic su install_acs_64.

Per installare la versione a 32 bit per tutti gli utenti su un PC multiutente, fare doppio clic su install_acs_32_allusers.

Per installare la versione a 64 bit per tutti gli utenti su un PC multiutente, fare doppio clic su install_acs_64_allusers.

Se si tratta di una nuova installazione, verranno presentate diverse richieste che aiuteranno a personalizzare l'installazione iniziale. Se viene visualizzato un accordo di licenza, accettarlo.

Se si tratta di un aggiornamento, le impostazioni precedenti verranno conservate.

Al termine dell'installazione, verrà visualizzato un messaggio di completamento.

Un'installazione per un singolo utente riuscita copierà l'applicazione in: 

    /Users/user_id/IBM/ClientSolutions/

Nota: la verisone iniziale di questi script copiava i file del prodotto in un percorso relativo a MyDocuments. La versione corrente copia i file del prodotto in un percorso basato sulle variabili di ambiente %HOMEDRIVE% e %HOMEPATH%. Quando si utilizzano questi script per aggiornare il prodotto, gli aggiornamenti si verificano nella prima ubicazione trovata.

Un'installazione multiutente riuscita copierà l'applicazione in: 

    /Users/Public/IBM/ClientSolutions/

2.1 Avvio del prodotto su Windows

Fare doppio clic sull'icona "Access Client Solutions" sul desktop.

3.0 Installazione per Mac

Individuare l'archivio zip o il CD del prodotto utilizzando Finder.

Se si utilizza l'archivio zip del prodotto, estrarne il contenuto.
Suggerimento: se il prodotto è stato appena scaricato, cercare in Downloads. Se il contenuto dell'archivio zip non è stato estratto, estrarlo facendo doppio clic su esso.

Individuare la cartella Mac_Application. Fare doppio clic su install_acs

Se si tratta di una nuova installazione, verranno presentate diverse richieste che aiuteranno a personalizzare l'installazione iniziale. Se viene visualizzato un accordo di licenza, accettarlo.

Se si tratta di un aggiornamento, le impostazioni precedenti verranno conservate.

Al termine dell'installazione, verrà visualizzato un messaggio di completamento.

Una installazione riuscita copierà l'applicazione nella cartella Applicazioni.

Se l'installazione è stata avviata da Finder, è stata visualizzata una finestra del terminale per eseguire l'installazione. Una volta completata l'installazione, è possibile chiudere la finestra del terminale.

Nota tecnica:
In base alle impostazioni di sicurezza e alla versione di OS X che si sta utilizzando, è possibile che venga visualizzata una finestra a comparsa che indica che install_acs non è autorizzato all'esecuzione. Se ciò dovesse accadere, sarà necessario autorizzare install_acs all'esecuzione utilizzando uno dei seguenti metodi:

  • Se viene visualizzata un'opzione per aprire install_acs, selezionarla.
  • Fare clic con il tasto destro del mouse su install_acs e selezionare "Open".
    Quando viene visualizzata una finestra a comparsa, selezionare "Open".
  • Se nessuna delle opzioni sopra funziona, andare a:
    System Preferences...
    Security & Privacy
    Se è disponibile un'opzione per aprire "IBM i Access Client Solutions", selezionarla. Altrimenti, sarà necessario modificare le impostazioni nel separatore General per consentire l'esecuzione ad altre applicazioni. Dopo aver avviato "IBM i Access Client Solutions" per la prima volta, è possibile ripristinare le impostazioni di sicurezza.

3.1 Avvio del prodotto su Mac

Utilizzare Finder per andare ad Applications.

Individuare "IBM i Access Client Solutions" e fare doppio clic su esso per avviare il prodotto.

Nota tecnica:
In base alle impostazioni di sicurezza e alla versione di OS X che si sta utilizzando, è possibile che venga visualizzata una finestra a comparsa che indica che l'applicazione non è autorizzata all'esecuzione. Se ciò dovesse accadere, sarà necessario autorizzare l'applicazione all'esecuzione utilizzando uno dei seguenti metodi:

  • Se viene visualizzata un'opzione per aprirla, selezionarla.
  • Fare clic con il tasto destro del mouse sull'icona "IBM i Access Client Solutions" e selezionare "Open".
    Quando viene visualizzata una finestra a comparsa, selezionare "Open".
  • Se nessuna delle opzioni sopra funziona, andare a:
    System Preferences...
    Security & Privacy
    Se è disponibile un'opzione per aprire "IBM i Access Client Solutions", selezionarla. Altrimenti, sarà necessario modificare le impostazioni nel separatore General per consentire l'esecuzione ad altre applicazioni. Dopo aver avviato "IBM i Access Client Solutions" per la prima volta, è possibile ripristinare le impostazioni di sicurezza.
Note tecniche:
  • Se viene richiesto di installare "Java 6 SE runtime", non è necessario installare Java 6. Installare Java 8 e provare ad avviare nuovamente il prodotto.
  • Quando si installa Java su Mac, selezionare un JDK invece di un JRE. Il JRE per Mac installa Java solo per il browser. Non installa Java per altre applicazioni. L'installazione del JDK renderà Java disponibile per altre applicazioni come IBM i Access Client Solutions.

4.0 Installazione per Linux

Individuare l'archivio zip o il CD del prodotto.

Se si utilizza l'archivio zip del prodotto, estrarne il contenuto.
Suggerimento: da una finestra del terminale, utilizzare il comando unzip.

Individuare la cartella Linux_Application.

Poiché è necessario disporre dell'autorizzazione root per utilizzare lo script di installazione, utilizzare sudo per eseguire lo script di installazione da una finestra del terminale. Ad esempio:

Se si dispone di un'architettura a 32, utilizzare il seguente comando.

    sudo ./install_acs_32
Se si dispone di un'architettura a 64, utilizzare il seguente comando.
    sudo ./install_acs_64

Se si tratta di una nuova installazione, verranno presentate diverse richieste che aiuteranno a personalizzare l'installazione iniziale. Se viene visualizzato un accordo di licenza, accettarlo.

Se si tratta di un aggiornamento, le impostazioni precedenti verranno conservate.

Al termine dell'installazione, verrà visualizzato un messaggio di completamento.

Una installazione riuscita copierà l'applicazione in:

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Avvio del prodotto su Linux

Cercare "IBM i Access Client Solutions" nella cartella delle applicazioni o utilizzare la richiesta comandi immettendo "IBM i Access Client Solutions".

5.0 Installazione aggiornamenti

Quando una nuova versione diventa disponibile, è possibile seguire le istruzioni di installazione in questo documento per applicare gli aggiornamenti. Se viene richiesto di sovrascrivere i file esistenti, rispondere "Sì a tutti".

Se si stanno utilizzando gli script install_acs per Windows, Mac o Linux, il file AcsConfig.properties non verrà aggiornato, così qualsiasi modifica di configurazione personalizzata apportata a tale file non verrà persa.

6.0 Ulteriori informazioni

Per ulteriori informazioni sulla personalizzazione del prodotto o se si incontrano problemi durante l'installazione o l'avvio del prodotto, consultare il documento Introduzione che si trova nella cartella Documentation.

7.0 Disinstallazione

Per rimuovere i file dell'applicazione copiati nelle ubicazioni sopra indicate dallo script di installazione, utilizzare lo script di disinstallazione specifico della piattaforma, presente nella stessa cartella dello script di installazione: 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
