1.0 소개

이 안내서는 단일 사용자용으로 로컬 디스크 드라이브에 이 제품을 배치하는 방법에 대한 지침을 제공합니다.

다음을 포함하여 이 제품을 배치하는 방법은 여러가지입니다.

  • 여러 사용자가 액세스할 수 있는 공유 위치
    IBM i Access Client Solutions: 손쉬운 사용자 정의 및 배치를 참조하십시오.
  • 휴대용 매체(예: CD, DVD, USB 플래시 드라이브)를 포함하여 사용자가 선택한 위치로 또는 로컬 디스크 드라이브의 경로로 zip 아카이브를 압축 해제합니다. 문서 폴더에 있는 시작하기 문서의 설치 섹션을 참조하십시오.
  • 같은 구성을 다중 사용자에게 전파하는 구성 설정 자동으로 가져오기와 같은 옵션은 시작하기에 있는 기타 배치 옵션을 참조하고 IBM i에서 직접 명령행 플러그인 사용에 대해서는 기본 IBM i 배치를 참조하십시오.

단일 사용자용으로 로컬 디스크 드라이브에 이 제품을 배치하려면 이 문서의 나머지 섹션을 참조하십시오.

2.0 Windows용 설치

제품 zip 아카이브 또는 제품 CD를 찾으십시오.

제품 zip 아카이브를 사용하는 경우에는 zip 아카이브의 컨텐츠를 추출하십시오.
제안사항: Windows 탐색기를 사용하여 zip 아카이브를 선택한 후 마우스 오른쪽 버튼으로 클릭하고 "모두 추출..."을 선택하여 이를 수행할 수 있습니다.

Windows 탐색기를 사용하여 제품의 Windows_Application 폴더를 찾으십시오.

현재 사용자에 대해 32비트 버전을 설치하려면 install_acs_32를 두 번 클릭하십시오.

현재 사용자에 대해 64비트 버전을 설치하려면 install_acs_64를 두 번 클릭하십시오.

다중 사용자 PC의 모든 사용자에 대해 32비트 버전을 설치하려면 install_acs_32_allusers를 두 번 클릭하십시오.

다중 사용자 PC의 모든 사용자에 대해 64비트 버전을 설치하려면 install_acs_64_allusers를 두 번 클릭하십시오.

새로 설치하는 경우, 초기 설치를 사용자 정의하는 데 도움이 되는 여러 프롬트가 표시됩니다. 라이센스 계약이 표시되면 승인하십시오.

업그레이드인 경우 이전 설정값은 유지됩니다.

설치가 완료되면 완료 메세지가 표시됩니다.

단일 사용자 설치에서는 애플리케이션을 다음 위치로 복사합니다. 

    /Users/user_id/IBM/ClientSolutions/

주: 이러한 스크립트의 초기 버전에서는 제품 파일을 MyDocuments의 상대 경로로 복사했습니다. 현재 버전에서는 제품 파일을 환경 변수 %HOMEDRIVE% 및 %HOMEPATH%를 기반으로 하는 경로로 복사합니다. 이러한 스크립트를 사용하여 제품을 갱신하는 경우 갱신은 처음 찾은 위치에서 발생합니다.

다중 사용자 설치에서는 애플리케이션을 다음 위치로 복사합니다. 

    /Users/Public/IBM/ClientSolutions/

2.1 Windows에서 제품 시작

데스크탑에서 "Access Client Solutions" 아이콘을 두 번 클릭하십시오.

3.0 Mac용 설치

파인더를 사용하여 제품 zip 아카이브 또는 CD를 찾으십시오.

제품 zip 아카이브를 사용하는 경우에는 zip 아카이브의 컨텐츠를 추출하십시오.
제안사항: 방금 제품을 다운로드한 경우에는 다운로드를 확인하십시오. Zip 아카이브의 컨텐츠가 추출되지 않은 경우에는 zip 아카이브를 두 번 클릭하여 zip 아카이브의 컨텐츠를 추출할 수 있습니다.

Mac_Application 폴더를 찾으십시오. install_acs를 두 번 클릭하십시오.

새로 설치하는 경우, 초기 설치를 사용자 정의하는 데 도움이 되는 여러 프롬트가 표시됩니다. 라이센스 계약이 표시되면 승인하십시오.

업그레이드인 경우 이전 설정값은 유지됩니다.

설치가 완료되면 완료 메세지가 표시됩니다.

설치가 완료되면 애플리케이션 폴더에 애플리케이션이 복사됩니다.

파인더에서 설치를 시작한 경우 단말기 창이 열리며 설치를 수행합니다. 설치가 완료되면, 단말기 창을 닫을 수 있습니다.

기술적 참고사항:
사용 중인 OS X의 버전과 사용자의 보안 설정값에 따라 install_acs를 실행할 권한이 없음을 표시하는 팝업 창이 표시될 수 있습니다. 이런 경우 다음 방법 중 하나를 사용하여 install_acs 실행 권한을 부여해야 합니다.

  • install_acs "열기" 옵션이 표시되면 선택하십시오.
  • install_acs를 마우스 오른쪽 버튼으로 클릭한 후 "열기"를 선택하십시오.
    팝업 창이 표시되면 "열기"를 선택하십시오.
  • 위 옵션 중 어느 것도 작동하지 않으면 다음으로 이동하십시오.
    시스템 기본설정...
    보안 & 개인정보처리방침
    "IBM i Access Client Solutions" 열기 옵션이 있으면 선택하십시오. 그렇지 않으면 일반 탭에서 설정을 조정하여 기타 애플리케이션을 실행할 수 있도록 해야 합니다. "IBM i Access Client Solutions"가 처음 시작되면 보안 설정을 다시 변경할 수 있습니다.

4.0 Linux용 설치

제품 zip 아카이브 또는 제품 CD를 찾으십시오.

제품 zip 아카이브를 사용하는 경우에는 zip 아카이브의 컨텐츠를 추출하십시오.
제안사항: 단말기 창에서 unzip 명령을 사용하십시오.

Linux_Application 폴더를 찾으십시오.

설치 스크립트를 사용할 루트 권한이 필요할 것이므로 sudo를 사용하여 단말기 창에서 설치 스크립트를 실행하십시오. 예를 들면, 다음과 같습니다.

32비트 구조인 경우 다음 명령을 사용하십시오.

    sudo ./install_acs_32
64비트 구조인 경우 다음 명령을 사용하십시오.
    sudo ./install_acs_64

새로 설치하는 경우, 초기 설치를 사용자 정의하는 데 도움이 되는 여러 프롬트가 표시됩니다. 라이센스 계약이 표시되면 승인하십시오.

업그레이드인 경우 이전 설정값은 유지됩니다.

설치가 완료되면 완료 메세지가 표시됩니다.

설치가 완료되면 애플리케이션이 다음 위치로 복사됩니다. 

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Linux에서 제품 시작

애플리케이션 폴더에서 또는 수퍼 키를 사용한 후 "IBM i Access Client Solutions"를 입력하여 "IBM i Access Client Solutions"를 탐색합니다.

5.0 업데이트 설치

새 버전을 사용할 수 있으면, 이 문서의 설치 지침에 따라 업데이트를 적용하십시오. 기존 파일을 오버레이하도록 프롬트되면 "모두 예"로 응답하십시오.

Windows, Mac 또는 Linux용 install_acs 스크립트를 사용하는 경우에는 AcsConfig.properties 파일이 갱신되지 않으므로 해당 파일에 대해 작성된 사용자 정의 구성 변경사항이 유실되지 않습니다.

6.0 자세한 정보

제품 사용자 정의에 대한 추가 정보 또는 제품 설치 또는 시작 도중 문제를 발견했으면 Documentation 폴더에 있는 시작하기 문서를 참조하십시오.

7.0 설치 제거

설치 스크립트에 의해 위의 위치에 복사된 애플리케이션 파일을 제거하려면 설치 스크립트와 동일한 폴더에 있는 플랫폼별 제거 스크립트를 사용하십시오. 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
