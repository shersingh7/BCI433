1.0 Introdução

Este guia fornece instruções sobre como implementar este produto em uma unidade de disco local para um único usuário.

Há uma variedade de outras maneiras com que esse produto também pode ser implementado, incluindo:

  • uma localização compartilhada que pode ser acessada por múltiplos usuários.
    Veja IBM i Access Client Solutions: customização e implementação fáceis
  • a descompactação do archive zip em qualquer localização de sua escolha, incluindo mídia portátil (por exemplo, CD, DVD, unidade flash USB) ou qualquer caminho na unidade de disco local. Veja a seção Instalação do documento GettingStarted localizado na pasta Documentação
  • Veja Outras opções de implementação em GettingStarted para obter opções como Importar as definições de configuração automaticamente para propagar a mesma configuração para múltiplos usuários e Implementações nativas do IBM i para usar plug-ins da linha de comandos diretamente no IBM i.

Para implementar este produto na unidade de disco local de um único usuário, veja as seções restantes deste documento.

2.0 Instalação para Windows

Localize o archive zip do produto ou o CD do produto.

Se estiver usando o archive zip, extraia os conteúdos do archive zip.
Sugestão: isso pode ser feito usando o Windows Explorer para selecionar o archive zip, clicando com o botão direito nele e selecionando "Extrair todos...".

Usando o Windows Explorer, localize a pasta Windows_Application do produto.

Para instalar a versão de 32 bits para o usuário atual, clique duas vezes em install_acs_32.

Para instalar a versão de 64 bits para o usuário atual, clique duas vezes em install_acs_64.

Para instalar a versão de 32 bits para todos os usuários em um PC multiusuário, clique duas vezes em install_acs_32_allusers.

Para instalar a versão de 64 bits para todos os usuários em um PC multiusuário, clique duas vezes em install_acs_64_allusers.

Se esta for uma nova instalação, haverá vários prompts que ajudará a customizar a instalação inicial. Se aparecer um contrato de licença, aceite-o.

Se este for um upgrade, as configurações anteriores serão preservadas.

Quando a instalação for concluída, uma mensagem de conclusão será exibida.

Uma instalação de usuário único bem-sucedida copiará o aplicativo para: 

    /Users/user_id/IBM/ClientSolutions/

Observação: a versão inicial desses scripts copiou os arquivos do produto para um caminho relativo para MyDocuments. A versão atual copia os arquivos do produto para um caminho com base nas variáveis de ambiente %HOMEDRIVE% e %HOMEPATH%. Ao usar esses scripts para atualizar o produto, as atualizações ocorrerão no primeiro local encontrado.

Uma instalação multiusuário bem-sucedida copiará o aplicativo para: 

    /Users/Public/IBM/ClientSolutions/

2.1 Iniciando o produto no Windows

Dê um clique duplo no ícone "Soluções do cliente de acesso" na área de trabalho.

3.0 Instalação para Mac

Localize o archive zip do produto ou o CD do produto usando o Localizador.

Se estiver usando o archive zip, extraia os conteúdos do archive zip.
Sugestão: se tiver transferido o produto por download, veja em Downloads. Se os conteúdos do archive zip não tiverem sido extraídos, será possível extraí-los dando um clique duplo neles.

Localize a pasta Mac_Application. Clique duas vezes em install_acs

Se esta for uma nova instalação, haverá vários prompts que ajudará a customizar a instalação inicial. Se aparecer um contrato de licença, aceite-o.

Se este for um upgrade, as configurações anteriores serão preservadas.

Quando a instalação for concluída, uma mensagem de conclusão será exibida.

Uma instalação bem-sucedida copiará o aplicativo para a pasta Aplicativos.

Se você iniciou a instalação a partir do Localizador, uma janela do terminal foi aberta para executar a instalação. Quando a instalação for concluída, você pode fechar a janela do terminal.

Nota Técnica:
Dependendo de suas configurações de segurança e a versão do S.O X que estiver usando, você pode obter uma janela pop-up indicando que install_acs não está autorizado a executar. Se isso acontecer, será necessário autorizar o install_acs a executar usando um dos métodos a seguir:

  • Se você for apresentado com uma opção para "Abrir" o install_acs, então, selecione-a.
  • Clique como o botão direito do mouse em install_acs e selecione "Abrir".
    Quando apresentado com uma janela pop-up, selecione "Abrir".
  • Se nenhuma das opções acima funcionar, acesse:
    Preferências do Sistema...
    Segurança e Privacidade
    Se houver uma opção para Abrir "IBM i Access Client Solutions", selecione-a. Caso contrário, será necessário ajustar as configurações na guia Geral para permitir que outros aplicativos sejam executados. Após o "IBM i Access Client Solutions" ser iniciado pela primeira vez, você pode alterar suas configurações de segurança de volta.

3.1 Iniciando o produto no Mac

Use o Localizador para acessar os Aplicativos.

Localize "IBM i Access Client Solutions" e clique duas vezes nele para iniciar o produto.

Nota Técnica:
Dependendo de suas configurações de segurança e a versão do S.O. X que estiver usando, você pode obter uma janela pop-up indicando que o aplicativo não está autorizado a executar. Se isso acontecer, será necessário autorizar o aplicativo a executar um dos métodos a seguir:

  • Se você for apresentado com uma opção para "Abrir" o aplicativo, selecione-a.
  • Clique com o botão direito do mouse no ícone "IBM i Access Client Solutions" e selecione"Abrir".
    Quando apresentado com uma janela pop-up, selecione "Abrir".
  • Se nenhuma das opções acima funcionar, acesse:
    Preferências do Sistema...
    Segurança e Privacidade
    Se houver uma opção para Abrir "IBM i Access Client Solutions", selecione-a. Caso contrário, será necessário ajustar as configurações na guia Geral para permitir que outros aplicativos sejam executados. Após o "IBM i Access Client Solutions" ser iniciado pela primeira vez, você pode alterar suas configurações de segurança de volta.
Nota Técnica:
Se você for solicitado a instalar o "tempo de execução Java 6 SE", não será necessário instalar o Java 6. Instale o Java 8 e, em seguida, tente iniciar o produto novamente.

4.0 Instalação para Linux

Localize o archive zip do produto ou o CD do produto.

Se estiver usando o archive zip, extraia os conteúdos do archive zip.
Sugestão: Em uma janela do terminal, use o comando unzip.

Localize a pasta Linux_Application.

Como você precisará ter autoridade de administrador para usar o script de instalação, use sudo para executar o script de instalação a partir de uma janela do terminal. Por exemplo:

Se você estiver em uma arquitetura de 32 bit bits, use o comando a seguir:

    sudo ./install_acs_32
Se você estiver em uma arquitetura de 64 bit bits, use o comando a seguir:
    sudo ./install_acs_64

Se esta for uma nova instalação, haverá vários prompts que ajudará a customizar a instalação inicial. Se aparecer um contrato de licença, aceite-o.

Se este for um upgrade, as configurações anteriores serão preservadas.

Quando a instalação for concluída, uma mensagem de conclusão será exibida.

Uma instalação bem-sucedida copiará o aplicativo para:

    /opt/ibm/iAccessClientSolutions/
    /usr/share/applications/

4.1 Iniciando o produto no Linux

Procure por "IBM i Access Client Solutions" na pasta de aplicativos ou usando a Super chave e, em seguida, digitando "IBM i Access Client Solutions".

5.0 Instalando Atualizações

Quando uma nova versão estiver disponível, você pode seguir as instruções de instalação neste documento para aplicar as atualizações. Se você for solicitado a sobrepor os arquivos existentes, responda com "Sim para todos".

Se estiver usando os scripts install_acs scripts para Windows, Mac ou Linux, o arquivo AcsConfig.properties não será atualizado para que qualquer mudança feita no arquivo na configuração customizada não seja perdida.

Para obter informações adicionais sobre customização do produto ou se tiver problema ao instalar ou iniciar o produto, veja o documento GettingStarted localizado na pasta Documentação.

7.0 Desinstalação

Para remover os arquivos de aplicativos que foram copiados para os locais acima pelo script de instalação, use o script de desinstalação específico de plataforma localizado na mesma pasta que o script de instalação: 

    Windows_Application/uninstall_acs.js
    Mac_Application/uninstall_acs
    Linux_Application/uninstall_acs
